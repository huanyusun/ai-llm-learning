## 0. 一句话记住 Transformer

> Transformer = **Self-Attention + 位置编码 + 残差/LayerNorm + FFN**，完全不用 RNN/CNN。
> 核心运算只有一行：`Attention(Q,K,V) = softmax(QKᵀ/√d_k)·V`。

四个文件分别对应 Transformer 的四块拼图：

| 文件 | 解决的问题 | 对应论文小节 |
|------|-----------|------------|
| `attention.py` | 怎么让每个词"看"全句？ | §3.2.1 Scaled Dot-Product Attention |
| `02-多头注意力.py` | 一个头看不全，怎么多角度同时看？ | §3.2.2 Multi-Head Attention |
| `03-位置编码.py` | 自注意力不知道"谁先谁后"，怎么办？ | §3.5 PE + RoPE §3.2 |
| `04-mini_transformer_block.py` | 把零件拼成一整层？ | §3.1 残差+LN、§3.3 FFN |

## 1. 第 1 块：Self-Attention（单头，`attention.py`）

### 核心公式（必默写）

```
Attention(Q, K, V) = softmax( Q · Kᵀ / √d_k ) · V        [原论文 §3.2.1]
```

### 五步流程（对应 attention.py 的 5 个步骤）

| 步骤 | 操作 | 形状 | 直觉 |
|------|------|------|------|
| 0 | 输入 X | (n, d) | n 个词，每个 d 维 embedding |
| 1 | X·W_Q/W_K/W_V | (n, d)×3 | 投影成 Q/K/V（可学习参数） |
| 2 | Q·Kᵀ | (n, n) | 每个 query 对所有 key 的相似度 |
| 3 | ÷√d_k | (n, n) | 缩放，控方差 |
| 4 | softmax(最后一维) | (n, n) | 每行归一化成注意力权重 |
| 5 | attn·V | (n, d) | 用权重对 value 加权聚合 |

### Q/K/V 直觉（面试必答）

- **Q（Query，提问）**：当前词"想找什么样的信息"
- **K（Key，标签）**：每个词"能提供什么 / 怎么被找到"
- **V（Value，内容）**：每个词"实际携带的信息"
- 用 Q 跟所有 K 算相似度 → softmax 成权重 → 加权聚合 V → 得到融合了全句信息的新表示。

### ⚠️ 为什么除以 √d_k（高频追问，要会推导）

原论文 footnote 1：假设 q、k 的分量是均值 0、方差 1 的独立变量，点积 `q·k = Σ q_i·k_i` 的**方差 = d_k**。
- d_k 大 → 点积数值大 → softmax 进饱和区 → **梯度消失，训不动**。
- 除以 √d_k 把方差拉回 1，softmax 输入保持适度。
- **关键：这是控方差，不是归一化**（不改变相对大小排序，只改尺度）。

### 易错点
- softmax 在**最后一个维度（key 维）**做，不是 query 维——做反了意义完全不同。
- 自注意力本身**位置无关**（permutation equivariant），打乱输入输出只是对应打乱 → 必须靠位置编码（第 3 块）。

## 2. 第 2 块：Multi-Head Attention（`02-多头注意力.py`）

### 公式（必默写）

```
head_i  = Attention( Q·W_i^Q,  K·W_i^K,  V·W_i^V )         [原论文 §3.2.2]
MHA(Q,K,V) = Concat(head_1, ..., head_h) · W^O
```

### 为什么需要多头

原论文原话："**With a single attention head, averaging inhibits this.**"
单头要对全句做加权平均，会把不同子空间的信息"摊平"。多头让模型在**不同表示子空间联合 attend**——像一个评审团，每个评委盯一个角度（语法/指代/搭配/长程依赖）。

### 关键工程细节（面试常考）

1. **每头降到 d_k = d_model/h**（论文 h=8, d_k=d_v=64, d_model=512）。
2. **Concat 在特征维（最后一维）**，不是序列维：8 头各贡献 64 维 → 512 维。
3. **总计算量 ≈ 单头满维**：每头降维抵消了头数增加（02 课实测 FLOPs 比值 = 1.000）。
4. **头数怎么选**：原论文 Table 3 行 A——单头比最优差 0.9 BLEU，但头太多也下滑（最优在 8 附近）。

### 02 课的代码验证
- 4 个头的注意力矩阵**各不相同** → 可视化"评审团各看一面"。
- FLOPs 对比：单头满维 768 vs 多头 4×d_k=2 也是 768 → 论文论断坐实。

## 3. 第 3 块：位置编码（`03-位置编码.py`）

### 为什么必须有

原论文 §3.5 原话："Since our model contains no recurrence and no convolution, in order for the
model to make use of the order of the sequence, we must inject some information about
the relative or absolute position." → 否则 "狗咬人" 和 "人咬狗" 对模型一样。

### 方案 A：绝对正余弦 PE（原论文 §3.5）

```
PE(pos, 2i)   = sin( pos / 10000^{2i/d_model} )
PE(pos, 2i+1) = cos( pos / 10000^{2i/d_model} )
```
- **加性注入**：加在 embedding 上（不是相乘！）。
- 选 sin/cos 的两个理由：① **可外推**到更长序列（学习式 PE 外推差）；② **PE(pos+δ) 是 PE(pos) 的线性函数** → 模型容易学到相对位置。
  - 03 课实测：求线性映射 M 使 PE(pos+δ) ≈ PE(pos)·M，**重构误差 ≈ 1e-16**（机器精度）。
- 消融（Table 3 行 E）：学习式 PE 结果几乎相同 → 选 sin/cos 主要为外推性，不是精度。

### 方案 B：RoPE 旋转位置编码（现代 LLM 默认，RoPE §3.2-3.3）

核心思想：**用旋转矩阵编码绝对位置，内积中自然出现相对位置依赖**。
- 把 d 维向量拆成 d/2 个 2D 子空间，每子空间按 θ_i = 10000^{-2i/d} 旋转角度 m·θ_i。
- 三个性质：① 可外推；② 长期衰减；③ **乘性注入**（旋转，不是相加）。

### 03 课的代码验证（两个核心性质都坐实）
- **相对位置不变性**：相同相对距离 (m−n) 的内积标准差 ≈ 3e-16（机器精度）→ RoPE 把绝对位置编码后，内积只剩相对位置。
- **长期衰减**：对随机 q/k 统计平均，平均 |内积| 随相对距离单调衰减。

### ⚠️ RoPE 易错点
- **只乘 Q/K，不乘 V！**（位置信息要进 QKᵀ 内积，value 不加位置）
- 每个二维子空间用**不同频率** θ_i，不是整向量同一角度。

### PE vs RoPE 一句话
| | 正余弦 PE | 学习式 PE | RoPE |
|---|---|---|---|
| 注入方式 | 加在 embedding | 加在 embedding | **乘（旋转）在 Q/K** |
| 显式相对位置 | 间接 | 否 | 是 |
| 外推 | 较好 | 差 | 好 |
| 现代主流 | 否 | 否 | **是（LLaMA/Qwen/DeepSeek）** |

## 4. 第 4 块：Transformer Block（`04-mini_transformer_block.py`）

### 一个 Decoder Block 的数据流（Post-LN，原论文 §3.1）

```
z1 = LayerNorm( x + MaskedMHA(x) )      # 子层1：masked 自注意力 + 残差 + LN
z2 = LayerNorm( z1 + FFN(z1) )          # 子层2：FFN + 残差 + LN
```

### 四个关键零件

1. **Masked Multi-Head Self-Attention**：causal mask 保证位置 i 只看 ≤i（自回归）。
   - 实现：softmax 输入的未来位置填 **−∞**（04 课用 -1e9 近似）。
   - 04 课验证：篡改位置 3（位置 1 的未来），位置 1 输出变化 = **0.00e+00** → mask 真生效。
2. **Position-wise FFN**（§3.3）：两层线性 + ReLU，`d_model → 4·d_model → d_model`。
   - 各位置共享参数，层与层不同；是模型**容量的主要来源**。
3. **残差连接**：`x + Sublayer(x)`，给信息一条"捷径"，缓解深层梯度消失。
4. **LayerNorm**（§3.1）：对**特征维**归一化（每个 token 的向量），不是 batch 维！

### ⚠️ 高频追问
- **Post-LN vs Pre-LN**：原论文是 Post-LN（需 warmup）；现代大模型多 Pre-LN（更稳、省 warmup）。
- **LayerNorm vs BatchNorm**：LN 跨特征维（适合变长序列），BN 跨 batch 维。
- **为什么 FFN 升 4×**：高维空间引入非线性，是模型容量的主要来源。
- **残差为什么能加**：所有子层输出维度都是 d_model，形状对齐才能相加——这也是 attention 输入输出同形状的意义。

### 为什么做 Decoder Block？
现代主流大模型（GPT/LLaMA/Qwen）都是 **Decoder-only**，核心就是带因果掩码的自注意力。
一个完整生成式 LLM = embedding + (RoPE + N 个 Block 堆叠) + 输出投影到词表。

## 5. 三大架构对比（基于原论文 §3.2.3 的注意力配置）

| 架构 | 注意力类型 | Mask | 典型任务 | 典型模型 |
|------|-----------|------|---------|---------|
| Encoder-only | 双向自注意力 | 无 causal | 理解（分类、NER） | BERT, RoBERTa |
| Decoder-only | causal 自注意力 | 有 causal | 生成（LM、对话） | GPT, LLaMA, Qwen |
| Encoder-Decoder | 双向 + causal + cross | decoder causal | seq2seq（翻译） | 原始 Transformer, T5 |

**为什么 Decoder-only 主流**：预训练统一（下一 token 预测）/ scaling 友好 / 无 cross-attn 开销 / 训练推理一致 / 涌现 ICL。

## 6. 面试速记卡（脱口而出版）

### 公式类（必须默写）
1. **Attention**：`softmax(QKᵀ/√d_k)V`
2. **√d_k 理由**：点积方差=d_k，大→softmax 饱和→梯度消失
3. **Multi-Head**：`head_i=Attention(QW_i^Q,KW_i^K,VW_i^V)`，`MHA=Concat(heads)·W^O`
4. **FFN**：`max(0,xW_1+b_1)W_2+b_2`，d_model→4·d_model→d_model
5. **正余弦 PE**：`sin(pos/10000^{2i/d})`, `cos(...)`
6. **残差+LN**：`LayerNorm(x+Sublayer(x))`

### 概念类（必须能解释）
7. Q/K/V：Query=提问，Key=标签，Value=内容，相似度→权重→加权 V
8. 多头意义：不同子空间联合 attend，单头会被平均抑制
9. causal mask：softmax 输入非法位置填 −∞（不是 0！）
10. RoPE 核心：旋转编码绝对位置，内积中只剩相对位置；只乘 Q/K 不乘 V
11. 位置编码必要性：自注意力本身 permutation equivariant

### 易错陷阱
- √d_k 缩放是**控方差不是归一化**
- softmax 在 **key 维（最后一维）** 做
- causal mask 填 **−∞ 不是 0**
- Multi-Head 每头**降维**（不是满维算 h 遍）
- RoPE **只乘 Q/K**
- 正余弦 PE 是**加在 embedding 上**（加性），RoPE 才是乘性
- LayerNorm 归一化**特征维**，不是 batch 维
