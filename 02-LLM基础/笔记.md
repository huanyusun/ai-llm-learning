## 2. 核心知识串联（理论 → 代码 → 面试）

### 2.1 Tokenizer（`01` 文件）
- **是什么**：把文本切成 token（子词单元）的工具；模型读 token，不读字符也不读词。
- **BPE**（GPT-2/BART/LLaMA 用）：以字节为基本符号，迭代合并【最频繁共现的相邻对】，选对标准是**频率**（[S1] §3.2）。
- **WordPiece**（BERT）：思想类似但选**似然增益最大的对**（[S1] §3.2）。
- **关键工程点**：词表训练前固定；**跨模型不通用**（GPT 的 token id 不能喂 LLaMA）；中文更费 token（同语义 token 数多 → 更贵 + 更早撑爆上下文窗口）。
- **面试高频**：BPE vs WordPiece 区别？为什么 LLaMA 用 Byte-level BPE？中文为什么费 token？

### 2.2 采样策略（`02` 文件，纯 numpy 实测）
对应 [S1] §6.1 三个公式：
- **Temperature**（式10）：`P(x_j) = exp(l_j/t) / Σexp(l_{j'}/t)`。t→0 贪心；t=1 默认采样；t→∞ 均匀。⚠ 是 logits **除以** t（不是乘）；t=0 是贪心**不是无采样**。
- **Top-k**：截断到概率最高的固定 k 个，重缩放后采样。k **固定**。
- **Top-p / nucleus**：累计概率 ≥ p 的**最小集合**，重缩放后采样。候选数**自适应** → 比 top-k 鲁棒。⚠ Top-p=1 **仍采样**，不是贪心。
- **生产经验**：事实问答/RAG 用低温度（0~0.3）；创意写作用高温度（0.7~1.0）；常配 top_p≈0.9 或 top_k≈40；三者可叠加（先 temperature 再 top-k/p）。

### 2.3 调用大模型（`03` 文件）
- **API 请求三件套**：`messages`（role: system/user/assistant）+ `model` + `采样参数`（temperature/top_p/max_tokens）。
- **本地 vs 云端**：ollama 提供 **OpenAI 兼容接口**（`http://localhost:11434/v1`），切换云端只需改 `base_url` + 加 API key。
- **工程纪律**：try/except + 友好降级提示，别让用户看裸 traceback；生产还要加重试、超时、限流、日志。

### 2.4 幻觉（`04` 文件）
- **定义**：生成看似合理但与事实/输入不符的内容。
- **标志数据**（[S2] §1，必背）：InstructGPT **21%** vs GPT-3 **41%** → 对齐让幻觉**约减半但非零**。
- **五大成因**（§6.2）：①训练目标（拟合似然而非真伪）②数据噪声 ③对齐讨好性（sycophancy）④知识截止 ⑤长上下文遗忘。
- **缓解 checklist**（§6.3）：RAG（grounding，⚠ 受检索质量制约）+ 低温度（⚠ 只压随机幻觉）+ 结构化 prompt + self-consistency + RLHF/DPO + red teaming + 让模型承认不知道。

## 3. 面试速查清单（连回理论库 §10）

1. **LLM 定义**：Transformer 语言模型，参数数百亿以上，海量文本训练；最小规模无共识，本综述以 >10B 为讨论范围（[S1] §2.1）。
2. **Scaling Law 两条**：KM（偏重增 N）vs Chinchilla（N、D 同比增长，`C≈6ND`，α=0.34,β=0.28）（[S1] §2.1）。
3. **训练三阶段**：预训练 → SFT → RLHF/DPO；InstructGPT 里 SFT = RLHF 第一步（[S1] §2.3, [S2] Fig.2）。
4. **RLHF 三步**：SFT → RM → PPO（[S2] Fig.2）。
5. **InstructGPT 标志结论**：1.3B InstructGPT 优于 175B GPT-3（参数差 100×）——「对齐 > 堆参数」（[S2] §1）。
6. **DPO**：闭式解把 RLHF 简化为分类损失，无需在线采样，稳定轻量（[S3] 摘要）。
7. **Tokenizer**：BPE（频率合并，GPT-2/LLaMA）vs WordPiece（似然增益，BERT）vs Unigram（[S1] §3.2）。
8. **few/one/zero-shot**：上下文窗口里塞 k 条示范；ICL 在前向传播内完成、无梯度更新（[S4] §2, Fig.1.1）。
9. **采样**：Temperature（t→0 贪心，t→∞ 均匀）、Top-k（固定候选数）、Top-p（累计概率自适应）（[S1] §6.1）。
10. **幻觉**：根因是概率拟合而非真伪；InstructGPT 21% vs GPT-3 41%；缓解靠 RLHF/RAG/低温度/red teaming（[S2] §1, [S1] §2.3, [WS]）。
11. **涌现**：阈值后突现、小模型没有；GPT-3 能 ICL、GPT-2 不能（[S1] §2.1）。
12. **KV Cache**：复用历史 K/V 避免重算，降延迟；代价是显存随序列线性增长（[WS]）。

## 4. 易错/陷阱（连回理论库 §11）

- 大 ≠ 对齐：做大模型不自动等于遵循用户意图（[S2] 摘要）。
- Scaling 不只看参数：KM 偏重 N 会「参数过剩、数据不足」，Chinchilla 主张 N、D 同比（[S1] §2.1）。
- SFT ≠ 传统 fine-tune：SFT 用多指令格式示范，目标是泛化（[S1] §2.3）。
- DPO ≠ 无奖励：仍有奖励概念，只是用策略参数化、闭式消掉显式 RM（[S3]）。
- BPE 选频率，WordPiece 选似然增益；tokenizer 跨模型不通用（[S1] §3.2）。
- ICL ≠ fine-tuning：无梯度更新，发生在单次前向（[S4] Fig.1.1）。
- **Temperature=0 是贪心不是无采样**；logits 是除以 t（[S1] §6.1）。
- **Top-p=1 仍采样**，不是贪心（[S1] §6.1）。
- 幻觉不可根除：RLHF 减半（21% vs 41%）但不为 0；RAG 受检索质量制约（[S2]/[WS]）。
- 涌现不保证：某些 LLM 可能不涌现（[S1] §2.1 脚注 1）。
- KV Cache 只在 decode 有用，请求间不共享（除非 prefix-sharing/paged-attention）（[WS]）。
- 公开 NLP 数据集 ≠ 真实使用：InstructGPT 明确指出二者不一致（[S2] §1）。

## 5. 三大「反直觉」压轴论点（面试收尾用，来自一手抓取）

1. **大不等于好**（[S2]）：1.3B InstructGPT 输出优于 175B GPT-3（参数差 100×）。对齐比堆参数更关键。
2. **参数过剩、数据不足是常态**（[S1] §2.1）：Chinchilla 律纠正了 KM 的「重参数轻数据」，业界转而大幅提高 token 数。
3. **策略即奖励**（[S3]）：DPO 证明在 KL 约束下，最优策略本身（相对参考模型的对数比）就等价于奖励函数——「Your LM is Secretly a Reward Model」。
