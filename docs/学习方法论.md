# 学习方法论：面试题 → 项目 → 学会

> 这是本项目的核心学习闭环。

---

## 🔄 工作流（每道题都这么走）

```
┌─────────────────────────────────────────────────────────┐
│  1. 出题：老师从面试题集锦里挑一道真实考题                │
│  2. 尝试：你回答（理论 0 也没关系，诚实说"不会"）         │
│  3. 讲解：老师用大白话 + 比喻讲清楚原理                   │
│  4. 实战：一起做一个最小项目，把理论"跑出来"              │
│  5. 沉淀：代码 + 笔记 + 你重新组织的答案，提交进本仓库     │
│  6. 复盘：你用自己的话再讲一遍（检验是否真懂）            │
└─────────────────────────────────────────────────────────┘
```

---

## 🎯 为什么用项目，而不是直接背八股？

- **背了讲不清**：面试官追问"为什么"，背答案是答不上来的
- **做过才真懂**：亲手跑一遍，理论就内化了，还能举一反三
- **项目即简历**：学完顺便有了 GitHub 项目，求职直接能用

---

## 📐 项目原则

- **最小可运行**：每个项目只解决一个核心概念，不贪多
- **能跑起来**：代码必须能运行，看到真实输出
- **注释详细**：理论 0 基础也看得懂每一行
- **从零搭建**：不用黑盒，每一步都讲为什么

---

## ✅ 每个模块的完成标准

一个模块算"学会"，要能：
1. ☐ 用自己的话把原理讲给小白听
2. ☐ 项目能独立跑起来并解释输出
3. ☐ 回答面试题的追问（why / how / 如果...怎么办）

---

## 🧰 每次学习的固定动作

每次开始新模块前：
1. 在对应模块文件夹建 `面试题.md`
2. 学习中产出 `笔记.md` + 项目代码
3. 学完更新 `README.md` 的进度表
4. `git commit` 并 `git push` 到 GitHub

> 一个 commit = 一次学习记录。坚持下来，GitHub 上的绿色贡献图就是你成长的证明 🌱
