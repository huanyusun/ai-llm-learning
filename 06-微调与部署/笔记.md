## 1. 微调（LoRA / SFT / DPO）

### 1.1 全参 vs PEFT

- **全参（FFT）**：所有参数都更新。显存 ≈ 基座×4（权重+梯度+Adam 的 m/v）。175B 全参 ≈ 1400GB，单机不可行。多任务部署要存 N 份完整副本。
- **PEFT**：只训少量额外参数。代表流派：
  - **Adapter**（加层，引入推理延迟）
  - **Prefix/Prompt-tuning**（占序列长度）
  - **LoRA**（低秩旁路，可合并→零延迟）✅ 主流
  - IA³、Bitfit（表达力弱）

### 1.2 LoRA 原理（必背公式）

```
W = W0 + ΔW = W0 + B·A
  B ∈ R^{d×r},  A ∈ R^{r×k},  r ≪ min(d,k)
前向：h = W0·x + (α/r)·B·A·x
```

- **可训练参数**：d·k → r·(d+k)，通常 <1%（论文 GPT-3 175B 上减 10000×）。
- **初始化**：A 随机高斯、B = **零** → 训练初始 `BA=0`，不破坏预训练（恒等启动）。
- **缩放 α/r**：固定 α≈2r 后基本不用再调学习率。
- **零推理延迟**：部署合并 `W = W0 + BA` 当普通层用。
- **多任务**：一份冻结基座 + N 个 MB 级 LoRA 适配器动态切换。
- **低秩为何够**：微调任务的「信息量」远小于预训练 → 权重增量在低维子空间（GPT-3 上 r=1/2 都够）。

> 见 `01-lora原理.py`：手写 `LoRALinear` 前向 + 链式法则反向 + 扫秩 r 的参数占比/误差曲线。

### 1.3 QLoRA（4-bit 基座上做 LoRA）

在 **冻结的 4-bit 量化基座** 上做 LoRA，单 48GB 卡微调 65B（780GB→<48GB）。三大创新：

1. **NF4（4-bit NormalFloat）**：对零均值正态分布信息论最优的 4-bit 数据类型（等概率分位数 + 精确表示 0）。
2. **Double Quantization**：对量化常数再做 8-bit 量化，省 ~0.37 bits/参数（65B 省 ~3GB）。
3. **Paged Optimizers**：NVIDIA 统一内存消除梯度检查点的显存尖峰。

单层前向：`Y = X·doubleDequant(W_NF4) + X·L1·L2`（存 NF4，算 BF16）。反向只更新 LoRA，4-bit 权重梯度不算。

> 关键经验：「LoRA on **all transformer layers** is critical to match 16-bit performance」；数据质量 >> 数量。

### 1.4 SFT（监督微调）

- 数据：`(instruction, output)` 或多轮 `messages`。**loss mask** 只在 assistant 输出 token 上算 loss。
- 通常用 LoRA/QLoRA，2.5k~10k 高质量样本即可显著改变行为。
- lr：LoRA 1e-4~3e-4；全参 1e-5~2e-5。epoch 1~3，过多→灾难性遗忘。
- 缓解遗忘：混通用数据、控 epoch、用 LoRA（冻结基座天然缓解）。

### 1.5 对齐：RLHF / DPO / GRPO

**RLHF（PPO）三阶段**（DPO 论文综述）：
1. SFT → π_SFT
2. 奖励建模：Bradley-Terry `p(y1≻y2|x)=σ(r(x,y1)−r(x,y2))`，二分类 NLL 训 `r_φ`。
3. RL：`max E[r_φ] − β·KL(π_θ‖π_ref)`，用 PPO（因离散生成不可微）。
- 痛点：4 个模型同显存（actor/ref/reward/critic）、on-policy 采样慢、PPO 敏感。

**DPO（直接偏好优化）** —— 「Your LM is secretly a reward model」：
```
L_DPO = −E[ log σ( β·log(π_θ(y_w|x)/π_ref(y_w|x)) − β·log(π_θ(y_l|x)/π_ref(y_l|x)) ) ]
```
- 由 KL 约束最优解的闭式形式反解出「奖励 = β·log(π_θ/π_ref)」，配分函数相消 → 不需奖励模型、不需 RL，纯二元交叉熵。
- π_ref = π_SFT；β∈[0.1,0.5]（小→偏离大、激进；大→强约束）。
- **陷阱**：naïve 损失（去掉动态重要性权重）会让模型退化 → 必须用完整 sigmoid 形式。

**GRPO（DeepSeek，PPO 变体）**：去掉 value/critic 网络；组内采样 G 个输出，用 `(r_i−μ)/σ` 做组相对优势替代 `A=R−V(s)`；奖励常来自**规则**（数学答案正确性）而非学习的模型 → RLVR/DeepSeek-R1。

选型：通用对齐/易实现 → DPO；可验证奖励（数学/代码）→ GRPO；最强上限/研究 → PPO-RLHF（最重）。

## 2. 量化（INT8/INT4、AWQ、GPTQ、GGUF）

核心：浮点 → 整数，`scale = max|W|/qmax`，`W_int = round(W/scale)`。
- 收益：FP32→INT8 省 4×、→INT4 省 8× 存储。
- 代价：round 误差；权重平滑分布时几乎无损。

| 方法 | 时机 | 数据 | 用途 | 生态 |
|------|------|------|------|------|
| bitsandbytes INT8/4 | PTQ/训练 | 校准 | QLoRA 训练、快速加载 | HF transformers |
| **NF4 (QLoRA)** | 训练 | 不需 | 微调基座 | bitsandbytes |
| **GPTQ-4bit** | PTQ 一次性 | ~128 校准 | 服务部署 | AutoGPTQ, vLLM |
| **AWQ-4bit** | PTQ 无反传 | 少量校准 | 服务部署（最快） | llm-awq, vLLM |
| GGUF (k-quants) | PTQ | - | CPU/Mac 端侧 | llama.cpp, ollama |
| FP8 | PTQ/训练 | - | 新 GPU (H100/Ada) | vLLM |

要点：
- **离群值是头号杀手**：`scale=max|W|/127` 被极端值撑大 → 其余值塌进几个刻度。LLM.int8() 混合精度保留离群列。
- **per-channel < per-tensor 误差**：每列独立 scale。AWQ/GPTQ 都在 per-channel 之上加保护。
- **AWQ**：保护 ~1% 按激活幅度判定的「显著权重」，per-channel scaling，硬件友好。
- **GPTQ**：逐列量化 + Hessian 二阶信息补偿剩余权重，一次性 PTQ。
- **GGUF k-quants**（Q4_K_M 等）：分块混合精度，关键层更高比特。

> 见 `02-量化演示.py`：手写量化/反量化、INT8/INT4 误差、离群值破坏、per-channel 修复。

典型路径：FP16 基座 → QLoRA 微调 → 合并 → GPTQ/AWQ INT4 量化 → vLLM/GGUF 部署。

## 3. 推理优化（KV Cache / Flash Attention / PagedAttention / 投机解码）

LLM 自回归逐 token 生成，**访存受限（memory-bound）**，优化都围绕「减少访存、提高显存利用率」。

### 3.1 KV Cache
- 生成第 t 个 token 时，前 t−1 个 token 的 K/V 不变 → 缓存复用，每步从 O(t) 降到接近 O(1)。
- 代价：KV cache 占显存巨大（13B 模型近 30%）；传统连续存放 → 严重碎片化。

### 3.2 FlashAttention（精确，非近似）
- **Tiling**：Q/K/V 分块，在 SRAM 内增量做 online softmax，**不把 N×N 矩阵物化到 HBM**。
- **Recomputation**：反向用前向存的 m、l 重算 attention，显存 O(N)（非 O(N²)）。
- HBM 访问从 `Ω(Nd+N²)` 降到 `O(N²d²/M)`，最多少 9×；GPT-2 上快 7.6×。
- 数学结果与标准 attention **逐位相同**。

### 3.3 PagedAttention / vLLM
- 痛点：传统系统 KV cache 连续存放，实测仅 **20.4%~38.2%** 显存被有效利用。
- 解法（借鉴 OS 虚拟内存分页）：KV 分成固定 token 数的 block，**非连续**存放，块表做逻辑→物理映射；按需分配；块共享（copy-on-write）支持并行采样/beam search。
- **Continuous Batching**：每步（iteration）重组 batch，结束的请求移出、新的插入；配合分页 KV，请求进出无需搬内存。
- vLLM 吞吐比 FasterTransformer/Orca 高 **2~4×**。
- 配套：chunked prefill、prefix caching、piecewise/full CUDA graphs、Multi-LoRA、量化全家桶（AWQ/GPTQ/GGUF/FP8）。

### 3.4 Speculative Decoding（投机解码）
- Draft（小快）投机生成候选 → Target（大慢）并行验证 → 接受一致的、从残差分布修正不一致的。
- **输出分布与仅用 target 采样完全相同**（无质量损失），典型 2~3× 加速。
- vLLM 支持 n-gram / suffix / EAGLE 等。

> 见 `04-推理优化演示.py`：手写带/不带 KV cache 的 attention，对比每步计算量与墙钟时间，验证数学等价。

## 5. 面试要点速记（带答案）

**Q：LoRA 公式与参数量？**
W=W0+BA，B∈R^{d×r}, A∈R^{r×k}, r≪min(d,k)。可训练从 d·k → r·(d+k)，通常 <1%（GPT-3 上减 10000×）。

**Q：LoRA 的 A/B 怎么初始化？**
A 高斯、B 零 → 初始 BA=0 不破坏预训练。前向加 α/r 缩放。

**Q：LoRA 为何无额外推理延迟？**
部署合并 W=W0+BA 存单一矩阵，对比 adapter（加深网络）/prefix-tuning（占序列长度）无延迟。

**Q：QLoRA 三大创新？**
NF4（对正态分布信息论最优 4-bit）、Double Quantization（量化常数再量化，省 0.37 bits/参数）、Paged Optimizers（统一内存消显存尖峰）。

**Q：QLoRA 怎么在 4-bit 上反传？**
存 NF4、算 BF16；用到权重时反量化到 BF16 做矩阵乘；反传只更新 LoRA，4-bit 权重梯度不算。

**Q：RLHF 三阶段 + 为何不稳定？**
SFT→奖励建模(BT)→RL(PPO+KL)。不稳来自 4 模型同显存、on-policy 采样、PPO 敏感、reward hacking。

**Q：DPO 损失 + 为何不需奖励模型？**
L=−E[log σ(β log(π_θ(y_w)/π_ref(y_w)) − β log(π_θ(y_l)/π_ref(y_l)))]。由 KL 约束最优解闭式形式反解出「奖励=β log(π_θ/π_ref)」，配分函数相消 → 直接用二元交叉熵。β 控 KL 强度，常用 0.1。

**Q：GRPO 比 PPO 省了什么？**
去 value 网络；组内 G 个采样用 (r_i−μ)/σ 做组相对优势；奖励常用规则（RLVR）。

**Q：FlashAttention 为何快？是近似吗？**
精确。Tiling 在 SRAM 做 online softmax 不物化 N×N；反向重算而非读中间矩阵。HBM 访问最多少 9×，快 7.6×，显存随序列线性。

**Q：PagedAttention 解决什么？**
KV cache 连续存放的碎片（仅 20.4%~38.2% 有效）。借鉴 OS 分页：KV 按 block 非连续存放 + 块表 + COW 共享。vLLM 吞吐 2~4×。

**Q：Continuous Batching？**
每步重组 batch，结束移出、新的插入；配合分页 KV 进出不搬内存。≠请求级 dynamic batching（前者是 token/iteration 级，吞吐更高）。

**Q：AWQ vs GPTQ？**
GPTQ：OBQ + 逐列 Hessian 二阶补偿，一次性 PTQ，需校准。AWQ：保护 ~1% 按激活幅度判定的显著权重，per-channel scaling，无二阶补偿、更硬件友好。都是 weight-only INT4。

**Q：Speculative Decoding 改变分布吗？**
不变。draft 猜 + target 验，接受/残差修正，数学上与仅用 target 采样完全相同，2~3×。

**Q：量化为何有效？**
权重平滑正态分布，用很少刻度就能很好近似。离群值是头号杀手（LLM.int8 混合精度）；per-channel < per-tensor。

## 6. 易错点 / 陷阱（高密度版）

1. LoRA target_modules：现代实践覆盖**全部线性层**（q/k/v/o/gate/up/down），不是只挂 q/v。
2. LoRA 前向是 `(α/r)·BAx`，不是 `BAx`；HF PEFT 的 `lora_alpha` 即 α。
3. LoRA 合并后难以同 batch 多任务（需不合并、动态选 LoRA 模块 → vLLM Multi-LoRA）。
4. DPO ≠「简单拉大好/压坏 logprob」：去掉动态重要性权重的 naïve 版会退化。
5. DPO 的 π_ref 要来自 π_SFT（或先用偏好回答做 MLE 初始化）。
6. FlashAttention ≠ 近似注意力（精确）；block-sparse 版才是近似。
7. INT4 在数学/长尾知识有可见退化；离群值大用 AWQ/GPTQ 而非朴素 round。
8. GPTQ 需要校准集（算 Hessian 依赖激活统计）。
9. KV cache 是显存大头（13B 近 30%）；长上下文用 PagedAttention + FP8 KV。
10. Continuous Batching（token 级）≠ Dynamic Batching（请求级）。
11. QLoRA 显存省在 **4-bit 基座**，不在梯度（7B 上 LoRA 参数仅 26MB，但输入梯度 567MB）。
12. Speculative Decoding 依赖 draft/target 分布相近，否则接受率低反而变慢。
13. 灾难性遗忘：纯窄域 SFT 丢通用能力 → 混通用数据、控 epoch、用 LoRA。
14. GGUF 的 Q4_K_M 不是均匀 4-bit（分块混合精度，关键层更高比特）。
