## ✅ 可靠性说明（来源透明可追溯）

| 标记 | 含义 | 模块 |
|---|---|---|
| 📕 | 基于**实体书**（出版社 ISBN）系统提炼 | RAG |
| 📄 | 基于**一手论文**(arXiv) + 官方文档 + 开源教材重建，每条标 `[来源 §x]` | 其余 5 模块 |

> 📄 模块重建方式：子代理实际 `curl`/WebFetch 抓取论文 HTML 全文（ar5iv.labs.arxiv.org）与官方文档原文，基于原文重写，关键论断/公式/数据后标注来源编号。**可追溯、可验证、不凭记忆编造**。
> 每个模块文件末尾有【**开源覆盖缺口**】：列出开源资源没讲透的点，可用书籍补充。

## 学习顺序建议（按依赖关系）

```
① Transformer   ← 地基，所有 LLM 的架构
     ↓
② LLM 基础      ← 训练/采样/幻觉（依赖 Transformer）
     ↓
③ Prompt 工程   ← 最常用，门槛低收益高
     ↓
④ RAG           ← 应用主力（最完整，13 章）
     ↓
⑤ Agent         ← 最热门（依赖 Prompt/RAG）
     ↓
⑥ 微调与部署    ← 进阶（依赖前面所有）
```

> 数学基础（线性代数/微积分）按需补，遇到不懂的查 [`../00-数学基础/`](../00-数学基础/)。

## 每个知识文件的结构

- **【参考资料来源】**：开头列出所有论文/文档（标题+arXiv号+URL）
- **正文**：知识点按「是什么 / 为什么 / 原理 / 流程」展开，关键处标来源
- **【面试考点】**：该模块高频面试题 + 详细答案
- **【易错/陷阱】**：常见误解与坑
- **【开源覆盖缺口】**：开源资源没讲透、可用书籍补充的点
