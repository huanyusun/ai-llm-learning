## §1 多模态学习概述

### 1.1 什么是多模态

- **模态（Modality）**：信息的表现形式，如文本、图像、音频、视频、表格等。
- **多模态学习**：让模型同时理解和处理多种模态信息的机器学习范式。
- **核心挑战**：
  - **表示（Representation）**：如何将不同模态映射到统一的表示空间？
  - **对齐（Alignment）**：如何建立不同模态元素之间的对应关系？
  - **融合（Fusion）**：如何有效组合多模态信息？
  - **转换（Translation）**：如何从一种模态生成另一种模态？

### 1.2 多模态的发展脉络

| 阶段 | 时间 | 代表工作 | 特点 |
|---|---|---|---|
| 早期融合 | ~2015 | CNN+RNN | 简单拼接特征 |
| 注意力融合 | 2015-2020 | VQA 模型 | 跨模态注意力 |
| 预训练对齐 | 2021 | CLIP、ALIGN | 大规模对比学习 |
| 多模态大模型 | 2023- | LLaVA、GPT-4V | 视觉编码器+LLM |

## §2 视觉编码器：ViT

### 2.1 ViT 核心思想 [V1]

**"An Image is Worth 16x16 Words"** — 把图像切成 patch，当作 token 序列送入 Transformer。

**流程**：
1. 输入图像 `H×W×C`（如 224×224×3）
2. 切成 `N = (H/P) × (W/P)` 个 patch（P=16 时，N=196）
3. 每个 patch 展平为向量，经线性投影得到 patch embedding
4. 前置一个可学习的 `[CLS]` token
5. 加上可学习的位置编码（1D，非 2D）
6. 送入 L 层 Transformer Encoder
7. `[CLS]` token 的输出作为图像的全局表示

### 2.2 ViT vs CNN

| 维度 | CNN | ViT |
|---|---|---|
| 归纳偏置 | 局部性 + 平移不变性（强先验） | 几乎无（靠数据学） |
| 小数据表现 | 好（先验帮助泛化） | 差（需大数据集） |
| 大数据表现 | 饱和 | 持续提升（scaling） |
| 全局建模 | 需堆叠多层 | 第一层就能全局注意 |
| 计算复杂度 | O(N)（卷积核固定） | O(N²)（自注意力） |

### 2.3 ViT 变体

- **DeiT**：数据高效 ViT，引入蒸馏 token，小数据也能训好。
- **Swin Transformer**：层级式 ViT，窗口注意力 + 移位窗口，计算更高效。
- **MAE**（Masked Autoencoder）：遮住 75% patch 让 ViT 重建，自监督预训练。

## §3 CLIP：对比语言-图像预训练

### 3.1 核心思想 [C1]

用**对比学习**将图像和文本映射到同一个向量空间，使匹配的图文对距离近、不匹配的距离远。

### 3.2 架构

```
图像 ──→ 图像编码器(ViT/ResNet) ──→ 图像向量 ──┐
                                                  ├──→ 余弦相似度矩阵
文本 ──→ 文本编码器(Transformer)  ──→ 文本向量 ──┘
```

### 3.3 训练目标：InfoNCE Loss

给定 batch 中 N 个图文对 `(I_i, T_i)`：
- 正样本：`(I_i, T_i)` 对角线上的 N 对
- 负样本：`(I_i, T_j)` 其中 `i≠j`，共 N²-N 对

Loss = 图像→文本的交叉熵 + 文本→图像的交叉熵（对称）

**关键**：batch size 越大，负样本越多，效果越好（CLIP 用 batch=32768）。

### 3.4 Zero-shot 分类

```
1. 准备类别文本：["a photo of a cat", "a photo of a dog", ...]
2. 用文本编码器得到每个类别的文本向量
3. 用图像编码器得到待分类图像的向量
4. 计算图像向量与所有类别文本向量的余弦相似度
5. 相似度最高的类别即为预测结果
```

不需要在目标数据集上微调，直接用预训练模型推理。

### 3.5 CLIP 的应用

| 应用 | 说明 |
|---|---|
| Zero-shot 分类 | 无需标注数据，文本描述即类别 |
| 图文检索 | 文搜图、图搜文 |
| Stable Diffusion | 用 CLIP 文本编码器引导图像生成 |
| 图像质量评估 | 计算生成图像与文本描述的对齐度 |

## §4 多模态大模型

### 4.1 核心架构

```
图像 → 视觉编码器(ViT) → 视觉特征 → 投影层(Linear/MLP) → 视觉 token
                                                                    ↓
文本 → Tokenizer → 文本 token ──────────────────────────────→ 拼接 → LLM → 输出
```

**三大组件**：
1. **视觉编码器**：通常用预训练的 ViT（如 CLIP-ViT-L/14），提取图像特征
2. **投影层**：将视觉特征映射到 LLM 的输入空间（维度对齐）
3. **LLM**：接收视觉 token + 文本 token，生成回答

### 4.2 LLaVA [L1]

**训练两阶段**：

| 阶段 | 数据 | 训练参数 | 目标 |
|---|---|---|---|
| 阶段1：预训练 | 595K 图文对 | 只训投影层（ViT+LLM 冻结） | 视觉-语言对齐 |
| 阶段2：指令微调 | 158K 视觉指令数据 | 投影层 + LLM（ViT 冻结） | 指令跟随能力 |

**关键设计**：
- 投影层只是一个线性层（LLaVA-1.0）或 MLP（LLaVA-1.5），参数量极小
- 阶段1 只需 ~1 小时训练（8×A100）
- 阶段2 解冻 LLM 是关键，让模型学会用视觉信息回答问题

### 4.3 主流多模态大模型对比

| 模型 | 视觉编码器 | LLM | 特点 |
|---|---|---|---|
| LLaVA-1.5 | CLIP-ViT-L/14 | Vicuna-13B | 开源，简洁高效 |
| GPT-4V | 未公开 | GPT-4 | 闭源，能力最强 |
| Gemini | 未公开 | Gemini | 原生多模态（非拼接） |
| Qwen-VL | ViT-bigG | Qwen-7B | 中文能力强 |
| InternVL | InternViT | InternLM | 开源最强之一 |

### 4.4 多模态能力边界

**能做的**：图像描述、VQA、OCR、图表理解、空间推理（部分）
**做不好的**：精确计数、空间关系推理（左右上下）、细粒度文字识别、视频长时理解

## §5 图像生成：Stable Diffusion

### 5.1 扩散模型基础

- **前向过程**：逐步给图像加高斯噪声，直到变成纯噪声
- **反向过程**：训练一个去噪网络（U-Net），从噪声逐步恢复图像
- **训练目标**：预测每一步加的噪声（noise prediction）

### 5.2 Stable Diffusion 架构 [SD1]

```
文本 → CLIP 文本编码器 → 文本嵌入
                              ↓ (cross-attention)
随机噪声 → U-Net (多步去噪) → 潜空间表示 → VAE 解码器 → 图像
```

**三大组件**：
1. **CLIP 文本编码器**：将文本描述编码为向量，通过 cross-attention 引导 U-Net
2. **U-Net**：在潜空间中执行去噪，是核心计算模块
3. **VAE**：编码器将图像压缩到潜空间，解码器将潜空间还原为图像

**为什么在潜空间**：像素空间维度太高（512×512×3 = 786K），潜空间压缩到 64×64×4 = 16K，计算量降低 ~50 倍。

### 5.3 关键技术

- **Classifier-Free Guidance (CFG)**：同时训有条件和无条件生成，推理时用两者差值放大文本引导效果
- **DDPM / DDIM**：不同的采样调度器，DDIM 可以用更少步数生成
- **ControlNet**：额外加入控制条件（边缘图、深度图、姿态），精确控制生成

## §6 面试高频考点汇总

### 必答题
1. CLIP 的训练目标和 zero-shot 原理
2. ViT 的 patch 机制和与 CNN 的区别
3. 多模态大模型的基本架构（ViT + 投影层 + LLM）

### 加分题
4. LLaVA 的两阶段训练策略
5. Stable Diffusion 的三大组件
6. 对比学习的 InfoNCE loss
7. 多模态模型的能力边界和局限性

### 易错点
- CLIP 不能生成文本，只能做匹配/检索
- ViT 的位置编码是 1D 的（不是 2D）
- 多模态大模型的投影层虽然简单但至关重要
- Stable Diffusion 在潜空间操作，不是像素空间
- GPT-4V 是闭源的，面试要能讲清开源方案
