## 0. 全局速览（30 秒建立心智模型）

在进入细节前，先用一段话把 Transformer "是什么、为什么、怎么算"串起来：

> Transformer 是一个**完全基于注意力机制**的序列模型，它**抛弃了 RNN 的顺序计算和 CNN 的局部感受野**，让序列中任意两个位置都能在 O(1) 步内直接交互（代价是 O(n²) 计算）`[原论文 摘要, §1]`。它的核心运算只有一行：`Attention(Q,K,V) = softmax(QK^T/√d_k)V` `[原论文 §3.2.1]`。因为自注意力本身不知道"谁先谁后"，所以需要位置编码（正余弦 `[原论文 §3.5]` 或 RoPE `[RoPE 摘要]`）注入顺序。整套架构由 Encoder（双向自注意力 + FFN）和 Decoder（causal 自注意力 + cross-attention + FFN）堆叠而成 `[原论文 §3]`，每层都包残差 + LayerNorm `[原论文 §3.1]`。面对长序列的 O(n²) 瓶颈，FlashAttention 用 tiling + 反向重算，在**不改数学结果**的前提下把 HBM 读写降下来 `[Flash 摘要]`。

**五个一句话结论**（面试可直接背）：

1. Transformer = Self-Attention + Positional Encoding + 残差/LayerNorm + FFN，不要 RNN/CNN `[原论文 摘要]`。
2. 注意力 = 用 Q 和所有 K 算相似度，softmax 成权重，加权 V `[原论文 §3.2]`。
3. 除 √d_k 是控方差防 softmax 饱和 `[原论文 §3.2.1]`。
4. Multi-Head = 多个子空间并行 attend，再拼回去 `[原论文 §3.2.2]`。
5. FlashAttention = 精确的 IO 优化（tiling + 重算），不是近似 `[Flash 标题]`。

## 1. 为什么需要 Transformer（动机与 RNN/LSTM 的缺陷）

### 1.1 RNN/CNN 的根本缺陷：顺序计算

原论文开篇即指出：循环模型（LSTM、GRU）将计算沿输入输出序列的符号位置展开，对齐到计算时间步——它逐步生成隐状态 `h_t`，作为 `h_{t-1}` 与位置 t 输入的函数。**这种内在的顺序性阻碍了训练样本内部的并行化**，序列越长越严重，因为内存约束限制了跨样本的批处理 `[原论文 §1]`。

> 原文："Recurrent models typically factor computation along the symbol positions of the input and output sequences... This inherently sequential nature precludes parallelization within training examples" `[原论文 §1]`。

卷积路径（ConvS2S、ByteNet）虽能在所有位置并行计算，但"关联任意两个输入/输出位置所需的操作数随两者距离增长（ConvS2S 线性增长、ByteNet 对数增长），使得远距离依赖更难学" `[原论文 §2]`。**Transformer 把这个操作数降到常数级**，代价是注意力加权平均带来的有效分辨率下降，由 Multi-Head 来弥补 `[原论文 §2]`。

### 1.2 Self-Attention 的三大优势（原论文 §4）

原论文用一张对比表（Table 1）从三个"desiderata（期望指标）"论证自注意力：

| 层类型 | 每层复杂度 | 顺序操作数 | 最大路径长度 |
|--------|-----------|-----------|-------------|
| Self-Attention | O(n²·d) | O(1) | O(1) |
| Recurrent (RNN) | O(n·d²) | O(n) | O(n) |
| Convolutional | O(k·n·d²) | O(1) | O(log_k(n)) |
| Self-Attention (restricted) | O(r·n·d) | O(1) | O(n/r) |

`[原论文 §4, Table 1]`（n=序列长，d=表示维度，k=卷积核宽，r=受限注意力邻域大小）

关键结论（直接来自原文）：

- 自注意力层**以常数级顺序操作**连接所有位置，而 RNN 需 O(n) 个顺序操作 `[原论文 §4]`。
- 当序列长 n **小于表示维度 d** 时，自注意力比 RNN 更快——这在机器翻译（BPE/word-piece 表示）中通常是成立的 `[原论文 §4]`。
- 单层卷积无法连接所有位置对，连续核需堆叠 O(n/k) 层，dilated conv 需 O(log_k(n)) 层 `[原论文 §4]`。
- 副作用：自注意力可产生更可解释的模型——不同 attention head 学到不同任务，部分呈现与句法/语义相关的行为 `[原论文 §4]`。

### 1.3 与 RNN/CNN 互补但被取代

> 原文："To the best of our knowledge, however, the Transformer is the **first transduction model relying entirely on self-attention** to compute representations of its input and output without using sequence-aligned RNNs or convolution." `[原论文 §2]`

## 2. Self-Attention 自注意力机制

### 2.1 注意力的统一语言：Query / Key / Value

原论文把注意力函数定义为：**将一个 query 和一组 key-value 对映射为一个输出**，输出是 values 的加权和，权重由 query 与对应 key 的"兼容函数（compatibility function）"计算 `[原论文 §3.2]`。

> 原文："An attention function can be described as mapping a query and a set of key-value pairs to an output, where the query, keys, values, and output are all vectors. The output is computed as a weighted sum of the values, where the weight assigned to each value is computed by a compatibility function of the query with the corresponding key." `[原论文 §3.2]`

d2l 用同一框架表述：给定查询 `q ∈ R^q` 和 m 个"键-值"对 `(k_i, v_i)`，注意力汇聚 f 就是 values 的加权和；权重由评分函数 `a(q,k_i)` 映射成标量、再经 softmax 得到 `[d2l 10.3, 式 10.3.1-10.3.2]`：

```
α(q, k_i) = softmax(a(q, k_i)) = exp(a(q,k_i)) / Σ_j exp(a(q,k_j))
f(q)      = Σ_i α(q, k_i) · v_i
```

### 2.2 Scaled Dot-Product Attention 公式

原论文的命名是 "Scaled Dot-Product Attention"。输入：维度为 `d_k` 的 queries 和 keys，维度为 `d_v` 的 values。计算 query 与所有 keys 的点积，每个除以 `√d_k`，再过 softmax 得到 values 上的权重 `[原论文 §3.2.1]`。

矩阵化公式（**本文最重要的一行**）：

```
Attention(Q, K, V) = softmax( Q · K^T / √d_k ) · V          [原论文 §3.2.1, 式 (1)]
```

- `Q ∈ R^{n×d_k}`：n 个 query 堆叠
- `K ∈ R^{n×d_k}`，`V ∈ R^{n×d_v}`
- `Q·K^T` 得到 `n×n` 的相似度矩阵
- softmax 在最后一个维度（每个 query 对所有 key 归一化）

d2l 给出逐向量版（式 10.3.4-10.3.5）：当 query/key 维度相同（都是 d）时，`a(q,k) = q^T k / √d`；批量版为 `softmax(Q K^T / √d) V ∈ R^{n×v}` `[d2l 10.3.3]`。

#### 维度推演（面试白板必会）

设 batch_size=b，序列长=n，d_k=d_v=64。逐步追踪每一步张量形状：

```
Q, K, V : (b, n, 64)
Q · K^T              : (b, n, 64) × (b, 64, n)  = (b, n, n)     # n×n 注意力分数
÷ √d_k (=8)          : (b, n, n)                                  # 缩放
+ mask（若有）       : (b, n, n)                                  # causal / padding
softmax(最后一维)    : (b, n, n)                                  # 每个 query 对所有 key 归一化
· V                  : (b, n, n) × (b, n, 64)  = (b, n, 64)       # 加权聚合 value
```

**面试口诀**：`Q(b,n,d)·K^T(b,d,n)→(b,n,n)`，softmax 后 `(b,n,n)·V(b,n,d)→(b,n,d)`，输出形状回到 `(b,n,d)`。注意力的"乘法链"两端形状相同（输入输出都是 n×d），这正是它能堆叠、能加残差连接的前提 `[原论文 §3.1]`。

### 2.3 为什么除以 √d_k（高频面试题，必须会推导）

原论文 footnote 1 给出严格推导：**假设 q 和 k 的分量是均值 0、方差 1 的独立随机变量**，则点积 `q·k = Σ_{i=1}^{d_k} q_i·k_i` 的均值为 0、**方差为 d_k** `[原论文 §3.2.1, footnote 1]`。

推论链条：

1. 点积方差随 d_k 线性增长 → 当 d_k 大时点积数值很大。
2. 大数值进入 softmax → 输出趋近 one-hot（最大值项 ≈1，其余 ≈0）。
3. softmax 落入**梯度极小的饱和区** → 训练信号消失 `[原论文 §3.2.1]`。

> 原文："We suspect that for large values of d_k, the dot products grow large in magnitude, **pushing the softmax function into regions where it has extremely small gradients**. To counteract this effect, we scale the dot products by √d_k." `[原论文 §3.2.1]`

除以 `√d_k` 后点积方差被拉回 1，softmax 输入保持适度，梯度正常。**记忆要点：除以 √d_k 是为了控制点积方差，防止 softmax 饱和、梯度消失**（不是归一化，不改变相对大小排序，只改变尺度）。

### 2.4 为什么用点积而不是加性注意力

原论文对比两种主流注意力（`[原论文 §3.2.1]`）：

- **加性注意力（Additive / Bahdanau）**：`a(q,k) = w_v^T tanh(W_q q + W_k k)`，用单隐层 MLP 计算兼容性 `[d2l 10.3.2, 式 10.3.3]`。适用于 q、k **长度不同**的场景 `[d2l 10.3.2]`。
- **点积注意力（Dot-Product / Multiplicative）**：与原论文方法相同，只差 `√d_k` 缩放因子 `[原论文 §3.2.1]`。

原论文选择点积的理由：两者理论复杂度相似，但**点积可用高度优化的矩阵乘法实现，实际更快、更省内存** `[原论文 §3.2.1]`。当 d_k 小时两者性能相近；d_k 大时**未缩放的点积不如加性注意力**，所以才加缩放 `[原论文 §3.2.1]`。

### 2.5 Self-Attention 三种使用方式（原论文 §3.2.3）

Transformer 在三个地方用 Multi-Head Attention `[原论文 §3.2.3]`：

1. **Encoder-Decoder 注意力**：Q 来自前一层 decoder，K/V 来自 encoder 输出 → decoder 每个位置可 attend 到输入序列所有位置 `[原论文 §3.2.3]`。
2. **Encoder 自注意力**：Q/K/V 全部来自 encoder 前一层 → 每个位置可 attend 到所有位置（双向）`[原论文 §3.2.3]`。
3. **Decoder 自注意力**：Q/K/V 全部来自 decoder 前一层，但加 **mask** 防止位置 attend 到后续位置（保持自回归性）；实现上把 softmax 输入中"非法连接"位置设为 -∞ `[原论文 §3.2.3]`。

> 原文："We implement this inside of scaled dot-product attention by **masking out (setting to −∞) all values in the input of the softmax** which correspond to illegal connections." `[原论文 §3.2.3]`

d2l 把这个通用化：**掩蔽 softmax 操作（masked softmax）**——任何超出有效长度的位置被掩蔽并置 0，实现上把被掩蔽位置填一个很大的负值（如 -1e6）再 softmax `[d2l 10.3.1]`。这是 padding mask 和 causal mask 的共同实现手法。

## 3. Multi-Head Attention 多头注意力

### 3.1 为什么需要多头

单头注意力的缺陷原论文点明："**With a single attention head, averaging inhibits this.**"——单头注意力对各位置做加权平均，会把不同子空间的信息"摊平"，抑制了模型在不同表示子空间联合 attend 信息的能力 `[原论文 §3.2.2]`。

### 3.2 多头公式与流程

原论文做法：不再做单次 d_model 维的注意力，而是把 Q/K/V 用**不同的、可学习的线性投影**投影 h 次到 `d_k / d_k / d_v` 维，每个投影并行做注意力，得到 h 个 `d_v` 维输出，**拼接后再做一次线性投影** W^O 得最终值 `[原论文 §3.2.2]`。

```
head_i = Attention( Q·W_i^Q,  K·W_i^K,  V·W_i^V )           [原论文 §3.2.2]
MultiHead(Q,K,V) = Concat(head_1, ..., head_h) · W^O
```

投影矩阵维度 `[原论文 §3.2.2]`：

- `W_i^Q ∈ R^{d_model × d_k}`
- `W_i^K ∈ R^{d_model × d_k}`
- `W_i^V ∈ R^{d_model × d_v}`
- `W^O   ∈ R^{(h·d_v) × d_model}`

### 3.3 每头降维 ≠ 计算量增加

原论文设置 h=8 个并行注意力头，每头 `d_k = d_v = d_model / h = 64`（d_model=512）。**由于每头维度降低，总计算量与单头满维注意力相近** `[原论文 §3.2.2]`。

> 原文："Due to the reduced dimension per head, the total computational cost is similar to that of single-head attention with full dimensionality." `[原论文 §3.2.2]`

### 3.4 多头的实验证据（消融）

原论文 Table 3 行 (A)：固定总计算量变头数——**单头比最佳设置差 0.9 BLEU，但头数过多质量也会下降** `[原论文 §6.2]`。这是面试常问"头数怎么选"的原文依据。

### 3.5 多头维度推演（与单头对比）

```
单头：  Q,K,V : (b, n, 512)
        QK^T  : (b, n, n)
        out   : (b, n, 512)

多头（h=8, 每头 d_k=d_v=64）：
  对每个 head i:
    Q·W_i^Q, K·W_i^K, V·W_i^V : (b, n, 64)        # 投影到子空间
    head_i = Attention(...)      : (b, n, 64)
  Concat(head_1..8)              : (b, n, 512)     # 沿特征维拼接
  · W^O                          : (b, n, 512)     # 最终投影
```

`[维度依据：原论文 §3.2.2 的投影矩阵定义]`

关键：**拼接发生在特征维（最后一维）而非序列维**，8 个头各贡献 64 维，合回 512 维。这保证了多头输出与单头输出形状一致，可无缝替换。

## 4. 位置编码（绝对正余弦 / RoPE 旋转位置编码）

### 4.1 为什么 Transformer 需要位置编码

原论文原文："Since our model contains no recurrence and no convolution, in order for the model to make use of the order of the sequence, we must inject some information about the relative or absolute position of the tokens." `[原论文 §3.5]`

即：自注意力本身是位置无关（permutation equivariant）的——打乱输入顺序，输出也对应打乱，但不改变每个 token 的表示。所以必须把位置信息注入到输入嵌入中。

### 4.2 原论文的正余弦绝对位置编码

原论文用不同频率的正弦/余弦函数 `[原论文 §3.5]`：

```
PE(pos, 2i)   = sin( pos / 10000^{2i/d_model} )
PE(pos, 2i+1) = cos( pos / 10000^{2i/d_model} )
```

- pos 是位置，i 是维度索引
- 每个维度对应一个正弦波，波长构成从 2π 到 10000·2π 的几何级数 `[原论文 §3.5]`

**为什么用正余弦而非学习式（两个动机，原文给出）**：

1. 假设它能"让模型容易学到按相对位置 attend"——因为对任意固定偏移 δ，`PE(pos+δ)` 可表示为 `PE(pos)` 的线性函数 `[原论文 §3.5]`。
2. **可外推到比训练时更长的序列**（学习式位置嵌入对未见长度泛化差）`[原论文 §3.5]`。

消融（Table 3 行 E）：用学习式位置嵌入替换正余弦，结果**几乎相同** `[原论文 §6.2]`。所以选正余弦主要是外推性而非精度。

### 4.3 RoPE 旋转位置编码（现代主流 LLM 默认方案）

#### 4.3.1 动机：把"绝对"和"相对"统一

RoPE 论文开宗明义：**用旋转矩阵编码绝对位置，同时在自注意力公式中自然引入显式的相对位置依赖** `[RoPE 摘要]`。

> 原文："the proposed RoPE encodes the absolute position with a rotation matrix and meanwhile incorporates the explicit relative position dependency in self-attention formulation." `[RoPE 摘要]`

RoPE 要解决的目标（§3.1）：希望 query `q_m` 与 key `k_n` 的内积 `⟨f(q,m), f(k,n)⟩` 只依赖词嵌入 `q, k` 和**相对位置 m−n**，即能写成 `g(q, k, m−n)` `[RoPE §3.1]`。

#### 4.3.2 二维情形的推导

2D 情形下，把词嵌入看成复数，RoPE 论文证明（§3.2、§3.4.1）：满足上述约束的解是**把仿射变换后的向量按位置索引的整数倍角度旋转** `[RoPE §3.2]`：

```
f(q, m) = (R_m · q̂)   （R_m 是角度为 mθ 的旋转矩阵，q̂ 为 q 的 2D 坐标）
```

> 原文："simply rotate the affine-transformed word embedding vector by amount of angle multiples of its position index and thus interprets the intuition behind Rotary Position Embedding." `[RoPE §3.2]`

#### 4.3.3 推广到 d 维（偶数）

把 d 维空间划分成 d/2 个 2D 子空间，利用内积的线性性组合起来 `[RoPE §3.2]`。等价于把向量相邻两两分组，每组按不同频率 θ_i 旋转：

```
R_Θ,m = diag( [cos(mθ_1), cos(mθ_1), cos(mθ_2), cos(mθ_2), ..., cos(mθ_{d/2}), cos(mθ_{d/2})] )
（配合交错正负 sin 实现 2×2 旋转块）
```

θ_i 通常取 `10000^{-2i/d}`（与原论文正余弦同源）`[RoPE §3.3]`。

#### 4.3.4 RoPE 的关键性质（论文给出的卖点）

1. **序列长度灵活（flexibility of sequence length）**：可外推 `[RoPE 摘要]`。
2. **随相对距离衰减的 token 依赖（decaying inter-token dependency）**：θ 设置为 10000 时有"长期衰减"性质——**内积随相对位置增大而衰减**，符合"距离远的 token 联系应更弱"的直觉 `[RoPE §3.3]`。
3. **乘性而非加性**：与传统加性位置编码不同，RoPE 通过旋转矩阵乘法注入位置 `[RoPE §3.2]`。

#### 4.3.5 RoPE vs 传统位置编码一句话

| 维度 | 正余弦绝对 PE | 学习式绝对 PE | RoPE |
|------|--------------|--------------|------|
| 注入方式 | 加在 embedding 上 | 加在 embedding 上 | 乘（旋转）在 Q/K 上 |
| 显式建模相对位置 | 间接（靠模型学） | 否 | 是 |
| 外推性 | 较好 | 差 | 好 |
| 现代主流 LLM | 否 | 否（受限） | **是（LLaMA/Qwen/等默认）** |

## 5. 整体架构：Encoder-Decoder、残差、LayerNorm、FFN

### 5.1 Encoder-Decoder 总体结构

原论文遵循经典 encoder-decoder 框架 `[原论文 §3]`：

- **Encoder**：把符号表示的输入序列 `(x_1,...,x_n)` 映射为连续表示 `z = (z_1,...,z_n)`。
- **Decoder**：给定 z，自回归地逐个生成输出符号 `(y_1,...,y_m)`——每一步消费之前生成的符号作为额外输入 `[原论文 §3]`。

Transformer 用堆叠的自注意力和逐位置全连接层分别实现 encoder 和 decoder `[原论文 §3]`。

### 5.2 Encoder 层（每层两个子层）

原论文 `[§3.1]`：encoder 由 N=6 个相同层堆叠。每层有两个子层：

1. **Multi-Head 自注意力**
2. **逐位置全连接前馈网络（FFN）**

每个子层外包**残差连接**（ResNet, He et al. 2016）+ **Layer Normalization**（Ba et al. 2016）：

```
output = LayerNorm( x + Sublayer(x) )        [原论文 §3.1]
```

为便于残差连接，所有子层和 embedding 层都输出维度 `d_model = 512` `[原论文 §3.1]`。

### 5.3 Decoder 层（每层三个子层）

decoder 同样 N=6 层，除 encoder 的两个子层外，**插入第三个子层**做对 encoder 输出的 Multi-Head Attention（即 §2.5 的 Encoder-Decoder 注意力）`[原论文 §3.1]`。同样残差 + LayerNorm。

decoder 自注意力子层被改造为**防止 attend 到后续位置**（causal mask），加上输出嵌入偏移一位（shifted right），保证位置 i 的预测只依赖位置 < i 的已知输出 `[原论文 §3.1]`。

### 5.4 Position-wise FFN

原论文 `[§3.3]`：每个 encoder/decoder 层除了注意力，还有一个逐位置独立、相同施加的全连接 FFN，包含两次线性变换，中间用 ReLU：

```
FFN(x) = max(0, x·W_1 + b_1)·W_2 + b_2        [原论文 §3.3, 式 (2)]
```

- 各位置共享同一组参数，但**层与层之间参数不同** `[原论文 §3.3]`。
- 等价描述：两个 kernel size=1 的卷积 `[原论文 §3.3]`。
- 输入输出维度 d_model=512，内层维度 d_ff=2048 `[原论文 §3.3, Table 3]`。

### 5.5 残差连接 + LayerNorm 的作用

- **残差连接**：缓解深层网络的梯度消失/退化，让信息有"捷径"直通。原论文每个子层都包残差 `[原论文 §3.1]`。
- **Layer Normalization**：对每个样本（这里指每个 token 的特征向量）做跨特征维度的归一化，稳定训练。原论文用 Post-LN（先残差再 LN）`[原论文 §3.1]`。

> 注意：现代实现常改为 **Pre-LN**（先 LN 再子层再残差），训练更稳、可省 warmup，但原论文是 Post-LN。这是论文未细讲的工程演化（见 §11 缺口）。

### 5.6 一层 Encoder 的完整数据流（白板默写版）

输入 `x : (b, n, 512)`：

```
# 子层 1：多头自注意力
z1 = LayerNorm( x + MultiHeadSelfAttention(x, x, x) )    # 双向，Q=K=V=x
# 子层 2：逐位置 FFN
z2 = LayerNorm( z1 + FFN(z1) )                           # FFN: 512→2048→512 (ReLU)
# 输出 z2 : (b, n, 512)，传给下一层
```
`[结构依据：原论文 §3.1, §3.2, §3.3]`

一层 Decoder 类似，但**三个子层**且自注意力带 causal mask，并多一个对 encoder 输出的 cross-attention：

```
# 子层 1：masked 多头自注意力
d1 = LayerNorm( x + MaskedMultiHeadSelfAttention(x) )          # causal
# 子层 2：cross-attention（Q=d1, K/V=encoder 输出）
d2 = LayerNorm( d1 + CrossAttention(d1, enc_out, enc_out) )
# 子层 3：FFN
d3 = LayerNorm( d2 + FFN(d2) )
```
`[结构依据：原论文 §3.1, §3.2.3]`

| 超参 | 值 | 出处 |
|------|-----|------|

| d_model | 512 | `[原论文 §3.1]` |

## 6. 三大架构对比：Encoder-only / Decoder-only / Encoder-Decoder

> 说明：原论文本身是 Encoder-Decoder（机器翻译任务）。Encoder-only / Decoder-only 是后续工作（BERT / GPT）对 Transformer 的剪裁，原论文未直接命名"三大架构"。本节架构对比的**机制部分来自原论文 §3.2.3 的三种注意力使用方式**，具体模型（BERT/GPT/T5）的归属由 WebSearch 补充验证。

### 6.1 三类的本质区别（基于原论文的注意力配置）

| 架构 | 用到的注意力类型 | Mask | 典型任务 | 典型模型 |
|------|-----------------|------|---------|---------|
| **Encoder-only** | 仅双向自注意力 | 无 causal mask（padding mask 可有） | 理解类：分类、NER、句对 | BERT, RoBERTa |
| **Decoder-only** | 仅带 causal mask 的自注意力 | 有 causal mask | 生成类：语言模型、对话、代码 | GPT 系列, LLaMA, Qwen |
| **Encoder-Decoder** | encoder 双向自注意力 + decoder causal 自注意力 + cross-attention | decoder 有 causal mask | 序列到序列：翻译、摘要 | 原始 Transformer, T5, BART |

机制依据全部来自原论文 §3.2.3 三种注意力使用方式 + decoder mask 说明：

- Encoder 自注意力：每个位置 attend 所有位置（**双向**）`[原论文 §3.2.3]`。
- Decoder 自注意力：causal mask，只能 attend 自身及之前位置（**单向**）`[原论文 §3.2.3]`。
- Cross-attention：Q 来自 decoder，K/V 来自 encoder `[原论文 §3.2.3]`。

### 6.2 为什么现代大语言模型多用 Decoder-only

> 这是论文未直接回答的问题，下面是业界共识的工程/理论原因（综合 WebSearch 验证 + 原论文机制推导）：

1. **训练范式统一**：Decoder-only 的目标就是"预测下一个 token"，与自回归语言建模天然契合，预训练数据只需无标注文本，可无限扩展。
2. **Scaling law 友好**：在相同参数/数据规模下，Decoder-only 展现出更平滑可预测的扩展性（GPT-3/Chinchilla 等经验）。
3. **无 cross-attention 开销**：Encoder-Decoder 多一套 cross-attention 参数和计算，Decoder-only 结构更简单、参数利用率更高。
4. **In-context learning 涌现**：纯 causal 结构 + 足够规模，催生 few-shot / instruction following 能力。
5. **机制上**：Decoder-only 的 causal mask 保证了训练时每个位置只看上文，与推理时（只能看上文）**训练/推理一致性（exposure bias 更小）**。

注意：Encoder-Decoder 在**封闭域翻译/摘要**、**有明确输入输出边界**的任务上仍有优势（T5 在多任务 benchmark 上长期强势）。

### 6.3 一句话区分

- **要双向理解（看全句再判别）→ Encoder-only**
- **要自回归生成（看上文接下文）→ Decoder-only**
- **要"读一段、写一段"（明确源/目标）→ Encoder-Decoder**

## 7. O(n²) 复杂度与 FlashAttention 优化

### 7.1 为什么是 O(n²)

原论文 Table 1 已写明 self-attention 每层复杂度 **O(n²·d)**（n=序列长，d=表示维度）`[原论文 §4, Table 1]`。直观来源：`Q·K^T` 产生 `n×n` 的注意力矩阵，要对所有 n² 个 (query, key) 对算点积并 softmax。

FlashAttention 论文开篇重申："Transformers are slow and memory-hungry on long sequences, since the time and memory complexity of self-attention are **quadratic in sequence length**." `[Flash §1]`

标准实现把 `S = Q·K^T ∈ R^{n×n}` 和 `P = softmax(S)` 都**物化（materialize）到 HBM**，占 O(n²) 内存 `[Flash §2.2]`。对 GPT-2 那样 n≈1K、d=64，n²d 项远大于 nd² `[Flash §2.2]`。

### 7.2 近似注意力的问题：只减 FLOP 不减内存访问

许多近似注意力（sparse、low-rank、linear）把计算降到线性/近线性，但"它们专注 FLOP 减少（而 FLOP 不一定与 wall-clock 速度相关），往往**忽略内存访问（IO）开销**，多数没有相对标准注意力的实际加速，未被广泛采用" `[Flash §1]`。

> 原文："these methods reduce the compute requirements to linear or near-linear... many of them **do not display wall-clock speedup against standard attention**... One main reason is that they focus on FLOP reduction... and tend to ignore overheads from memory access (IO)." `[Flash §1]`

### 7.3 FlashAttention 的核心思想：IO-aware

FlashAttention 抓住一个被忽视的原则——**让注意力算法感知 IO**，仔细管理快速 SRAM 与慢速 HBM 之间的读写 `[Flash 摘要, §1]`。

GPU 内存层级（Flash 论文实测数据，A100）`[Flash §2.1]`：

- HBM：40-80 GB，带宽 1.5-2.0 TB/s
- 片上 SRAM：每流处理器 192 KB，比 HBM 快一个数量级
- 现代 GPU 上**计算速度已超过内存速度**，Transformer 多数操作被内存访问瓶颈化 `[Flash §2.1]`

FlashAttention 的目标：**避免把 N×N 注意力矩阵在 HBM 反复读写** `[Flash §3]`。

### 7.4 两大技术：Tiling + Recomputation

#### 7.4.1 Tiling（分块）—— 在 SRAM 内增量做 softmax

把输入切成块（blocks），对输入块做**多次 pass**，**增量地完成 softmax 归约**（即 online softmax / 分块 softmax），无需访问整个输入即可算 softmax `[Flash 摘要, §3.1]`。

外循环逐块加载 Q 到 SRAM，内循环逐块加载 K、V，在片上完成"矩阵乘 → softmax（含掩码/dropout）→ 矩阵乘"全套运算，再写回 HBM `[Flash §3.1, Algorithm 1]`。这等价于把整套注意力**融合进一个 CUDA kernel** `[Flash §3.1]`。

> 原文："Tiling enables us to implement our algorithm in one CUDA kernel, loading input from HBM, performing all the computation steps (matrix multiply, softmax, optionally masking and dropout, matrix multiply), then write the result back to HBM." `[Flash §3.1]`

#### 7.4.2 Recomputation（重算）—— 反向时不存大矩阵

标准反向需要存 `S, P` 两个 n×n 矩阵算梯度。FlashAttention **反向时在片上从 Q/K/V 块重算** attention 矩阵，配合前向存的 softmax 归一化因子，比重读 HBM 中的中间矩阵更快 `[Flash 摘要, §3.1]`。

> 原文："We store the softmax normalization factor from the forward pass to quickly recompute attention on-chip in the backward pass, which is faster than the standard approach of reading the intermediate attention matrix from HBM." `[Flash §3.1]`

这是一种**选择性梯度检查点（selective gradient checkpointing）**——传统梯度检查点以速度换内存，而 FlashAttention 的重算虽然 FLOP 更多，但**因 HBM 访问大幅减少反而更快** `[Flash §3.1]`。

### 7.5 IO 复杂度定理（论文严格证明）

**Theorem 1**：FlashAttention 输出正确，FLOPs 与标准注意力相同（O(n²d)），**额外内存为 O(n)**（线性于序列长）`[Flash §3.1]`。

**Theorem 2（核心）**：设 n=序列长，d=头维，M=SRAM 大小（M > d）`[Flash §3.2]`：

| | HBM 访问次数 |
|---|---|
| 标准注意力（前向） | **Θ(n² + n·d)** |
| FlashAttention（前向） | **Θ(n²·d² / M)** |

反向：标准 **Θ(n²)**，FlashAttention **Θ(n²·d² / M)** `[Flash §3.2]`。

对典型 d=64-128、M≈100KB，`d²/M` 远小于 1，所以 FlashAttention 的 HBM 访问比标准实现**少很多倍（论文实测最多约 9×）** `[Flash §3.2]`。

**Proposition 3（下界）**：在 SRAM 大小 M 的某个取值范围内，**不存在能渐近改进 HBM 访问次数的精确注意力算法**——即 FlashAttention 在 IO 复杂度上是渐近最优的 `[Flash §3.2]`。

### 7.6 效果总结（论文实测）

- **更快**：注意力计算相对 PyTorch 标准实现加速最高 **7.6×**（GPT-2 上）`[Flash 摘要]`。
- **更省内存**：内存与序列长**线性**，而非 O(n²) `[Flash 摘要]`。
- **精确（exact）**：数学上与标准注意力等价，不是近似——这是它击败多数近似方法的关键 `[Flash 标题/摘要]`。
- 至序列长 512 时，FlashAttention 比任何已有方法都快且省内存；>1K 时部分近似方法（如 Linformer）可能更快，但 **block-sparse FlashAttention 比所有已知近似方法都快** `[Flash §1]`。

### 7.7 FlashAttention 一句话记忆

> **不改算法、只改内存访问模式**：用 tiling + online softmax 避免把 n×n 矩阵写入 HBM，用反向重算省掉中间矩阵存储——同样的精确结果，更少的 HBM 读写，线性的内存。

### 7.8 算法骨架（Algorithm 1 的伪代码还原）

来自 `[Flash §3.1, Algorithm 1]`，关键操作：

```
输入: Q, K, V 在 HBM，片上 SRAM 大小 = M
设块大小: Br × Bc（按 SRAM 容量定）
初始化: 输出 O, logsumexp L 在 HBM
把 Q 分成 Tr 个块 Q_j（每块 Br 行）
把 K,V 分成 Tc 个块 K_i, V_i（每块 Bc 行）
for j = 1..Tr:                          # 外循环：逐块 query
    把 Q_j 加载到 SRAM
    for i = 1..Tc:                       # 内循环：逐块 key/value
        把 K_i, V_i 加载到 SRAM
        在片上计算 S_ij = Q_j · K_i^T
        在片上做"增量 softmax"（用 running max + 重缩放，即 online softmax）
        在片上累积 O_j ← O_j + 校正后的 P_ij · V_i
    把最终 O_j 写回 HBM
返回 O
```

核心是 **online softmax**：softmax 原本需要一次性看到整行才能归一化（要全局 max 和 sum）；FlashAttention 通过维护"当前块的 running max 和 running sum"，每来一个新块就**重缩放已累积的结果**，从而不需要物化整个 n×n 矩阵就能得到精确 softmax `[Flash §3.1]`。这是它"精确且省内存"的数学根基。

### 7.9 标准 vs Flash 复杂度对照

| 维度 | 标准注意力 | FlashAttention |
|------|-----------|----------------|
| FLOPs | O(n²·d) | O(n²·d)（相同，反向略多） |
| 额外内存 | O(n²)（存 S 和 P） | **O(n)**（只存 O 和 logsumexp） |
| HBM 访问（前向） | Θ(n² + n·d) | **Θ(n²·d² / M)** |
| 是否精确 | 是 | **是（数学等价）** |
| 是否物化 n×n 矩阵 | 是 | **否** |

`[数据：Flash §2.2, §3.1, §3.2]`

**为什么 FLOP 相同却更快**：因为现代 GPU 上注意力是**内存绑定（memory-bound）**而非计算绑定——瓶颈是搬运 n×n 矩阵往返 HBM，而非算术 `[Flash §1, §2.1]`。Flash 把搬运量砍下来，wall-clock 就降了，哪怕反向多了点重算。

## 8. 训练机制要点（Embedding、Warmup、Dropout、Label Smoothing）

### 8.1 Embedding 与权重共享

- 用学习式 embedding 把输入/输出 token 转成 d_model=512 维向量 `[原论文 §3.4]`。
- decoder 输出经线性变换 + softmax 得下一 token 概率 `[原论文 §3.4]`。
- **两个 embedding 层与 pre-softmax 线性变换共享同一权重矩阵**（类似 Press & Wolf 2016）`[原论文 §3.4]`。
- embedding 层权重乘 `√d_model`（缩放，使 embedding 与位置编码量级匹配）`[原论文 §3.4]`。

### 8.2 优化器与 Warmup

- Adam 优化器，`β1=0.9, β2=0.98, ε=1e-9` `[原论文 §5.3]`。
- 学习率按公式变化：前 warmup_steps 步线性增长，之后按步数倒数平方根衰减 `[原论文 §5.3, 式 (3)]`：

```
lr = d_model^{-0.5} · min(step^{-0.5}, step · warmup_steps^{-1.5})
```

warmup_steps=4000。**warmup 是 Post-LN 架构稳定训练的关键**（Pre-LN 可大幅省 warmup——见 §11 缺口）。

### 8.3 正则化（三种）

1. **残差 Dropout**：作用在每个子层输出上（加到子层输入并归一化之前），以及 embedding+位置编码之和上，base model 比率 P_drop=0.1 `[原论文 §5.4]`。
2. **Label Smoothing**：值 ε_ls=0.1。**这会损害 perplexity（模型变得更不确定），但提升 accuracy 和 BLEU** `[原论文 §5.4]`。这是面试常考点：为什么 label smoothing 涨 BLEU 跌 PPL。
3. （base model 用 P_drop=0.1，big model 英法任务改用 0.3）`[原论文 §6.1]`。

### 8.4 训练成本与成果

- WMT 2014 EN-DE：big model **28.4 BLEU**，比此前最佳（含 ensemble）高 2+ BLEU `[原论文 摘要, §6.1]`。
- WMT 2014 EN-FR：单模型 **41.8 BLEU**，3.5 天 8 GPU `[原论文 摘要, §6.1]`。
- English constituency parsing：4 层 Transformer 在 WSJ 上 F1=91.3，半监督 92.7，证明**泛化性好** `[原论文 §6.3]`。

## 9. 面试考点速记

> 本节是为面试压缩的"脱口而出"卡片，每条都能对应到上方正文来源。

### A. 公式类（必须默写）

1. **Scaled Dot-Product Attention**：`Attention(Q,K,V) = softmax(QK^T/√d_k)V` `[原论文 §3.2.1]`。
2. **为什么 √d_k**：点积方差 = d_k，大 d_k 会使点积很大 → softmax 饱和 → 梯度消失；除 √d_k 把方差拉回 1 `[原论文 §3.2.1 footnote]`。
3. **Multi-Head**：`head_i = Attention(QW_i^Q, KW_i^K, VW_i^V)`，`MHA = Concat(head_1..h)·W^O`，h=8、d_k=d_v=64 `[原论文 §3.2.2]`。
4. **FFN**：`FFN(x)=max(0, xW_1+b_1)W_2+b_2`，d_model=512→d_ff=2048 `[原论文 §3.3]`。
5. **正余弦 PE**：`PE(pos,2i)=sin(pos/10000^{2i/d})`，`PE(pos,2i+1)=cos(pos/10000^{2i/d})` `[原论文 §3.5]`。
6. **LayerNorm + 残差**：`output = LayerNorm(x + Sublayer(x))` `[原论文 §3.1]`。

### B. 概念类（必须能解释）

7. **Q/K/V 直觉**：Query=当前 token 的"提问"，Key=其他 token 的"标签"，Value=其他 token 的"内容"；用 Q 与所有 K 算相似度，softmax 成权重，加权聚合 V `[原论文 §3.2]`。
8. **Multi-Head 的意义**：让模型在不同子空间联合 attend；单头会被平均抑制 `[原论文 §3.2.2]`。
9. **三种注意力使用**：encoder 自注意力（双向）/ decoder 自注意力（causal mask）/ cross-attention（Q 来自 decoder）`[原论文 §3.2.3]`。
10. **causal mask 实现**：softmax 输入中非法位置设 −∞ `[原论文 §3.2.3]`；d2l 的 masked softmax 把无效位置填大负值再 softmax `[d2l 10.3.1]`。
11. **正余弦 PE 选型理由**：可外推到更长序列 + 相对位置可线性表示；学习式结果几乎相同但外推差 `[原论文 §3.5, §6.2]`。
12. **RoPE 核心**：用旋转矩阵编码绝对位置，自注意力内积中自然出现相对位置依赖；有长期衰减、可外推 `[RoPE 摘要, §3.2, §3.3]`。

### C. 架构对比类

13. **三大架构**：Encoder-only（BERT，双向，理解）/ Decoder-only（GPT，causal，生成）/ Encoder-Decoder（T5，含 cross-attention，seq2seq）`[基于原论文 §3.2.3]`。
14. **为什么 Decoder-only 主流**：预训练统一（下一 token 预测）/ scaling 友好 / 无 cross-attn 开销 / 训练推理一致 / 涌现 ICL。
15. **O(n²) 来源**：QK^T 产生 n×n 矩阵，原论文 Table 1 明确 O(n²·d) `[原论文 §4]`。

### D. 工程优化类

16. **FlashAttention 是精确的不是近似**：与标准注意力数学等价 `[Flash 标题]`。
17. **FlashAttention 两大技术**：Tiling（分块 + online softmax，避免物化 n×n 到 HBM）+ Recomputation（反向重算，省中间矩阵）`[Flash §3.1]`。
18. **FlashAttention IO 复杂度**：标准 Θ(n²+nd)，Flash Θ(n²d²/M)，内存 O(n) `[Flash §3.2]`。
19. **为什么近似注意力不够**：只减 FLOP 不减 IO，wall-clock 不一定快 `[Flash §1]`。
20. **加性 vs 点积**：点积可用优化矩阵乘实现，更快更省内存；大 d_k 时点积需缩放才不输加性 `[原论文 §3.2.1]`。

### E. 训练类

21. **Label Smoothing 矛盾**：ε=0.1，BLEU/accuracy 涨但 PPL 跌 `[原论文 §5.4]`。
22. **Warmup**：前 4000 步线性增，之后倒数平方根衰减 `[原论文 §5.3]`。
23. **权重共享**：input embedding、output embedding、pre-softmax 线性层三处共享 `[原论文 §3.4]`。
24. **embedding ×√d_model**：让 embedding 量级与 PE 匹配 `[原论文 §3.4]`。

## 10. 易错 / 陷阱清单

> 这些是面试和实现中最容易答错、写错、理解错的点。

1. **√d_k 缩放的是方差，不是归一化**：它不改变 QK^T 元素的相对大小排序，只压尺度，目的是让 softmax 不饱和。说"归一化"是错的。

2. **softmax 维度易错**：注意力 softmax 在**最后一个维度（key 维度）**上做——每个 query 对所有 key 归一化。做反了会让每个 key 对所有 query 归一化，意义完全不同。

3. **causal mask 填的是 −∞ 不是 0**：填 −∞ 后 softmax 才归零；填 0 则 softmax 后仍有权重，信息泄漏。d2l 实现填 −1e6（数值稳定的近似）`[d2l 10.3.1]`。

4. **Multi-Head 不是"多算几遍同维度"**：每头维度降到 d_model/h，总计算量才与单头满维相当；若每头都保持 d_model，计算量会翻 h 倍 `[原论文 §3.2.2]`。

5. **RoPE 只乘 Q/K，不乘 V**：旋转施加在 query 和 key 上（因为位置信息要进 QK^T 内积），value 通常不加位置。实现里常见错误是给 V 也加。

6. **RoPE 在每个 2D 子空间用不同频率**：不是整向量用同一角度，而是相邻两两分组、各组 θ_i 不同（θ_i = 10000^{-2i/d}）`[RoPE §3.2]`。

7. **正余弦 PE 是加在 embedding 上，不是相乘**：原论文明确 "add positional encodings to the input embeddings" `[原论文 §3.5]`。RoPE 才是乘（旋转）。

8. **O(n²) 指 self-attention 单层**：不是整个模型；但它是长序列的主要瓶颈，且每层都有 `[原论文 §4]`。

9. **FlashAttention 不改变结果**：它是 IO 优化，不是近似算法；说"FlashAttention 是近似注意力"是错的（block-sparse 版本才是近似）`[Flash 标题/§1]`。

10. **Pre-LN vs Post-LN**：原论文是 Post-LN（LayerNorm 在残差外），需要 warmup；现代大模型多用 Pre-LN（LayerNorm 在子层内），更稳。论文未对比，面试常追问。

11. **Decoder 输入要 shifted right**：训练时 decoder 输入是目标序列右移一位，第一位是 <BOS>，保证位置 i 只看 < i `[原论文 §3.1]`。

12. **Self-attention 本身没有位置感知**：必须靠 PE/RoPE；打乱输入顺序，输出只是对应打乱——这是"为什么需要位置编码"的根因 `[原论文 §3.5]`。

13. **加性注意力用于 q/k 维度不同时**：点积要求 q/k 同维；维度不同用加性 `[d2l 10.3.2]`。Transformer 里 Q/K 同维所以用点积。

14. **embedding 权重 ×√d_model 的原因常被忽略**：是为了让 embedding 与 PE 数值量级匹配，否则 PE（量级 ~1）会被 embedding（量级小）淹没或反之 `[原论文 §3.4]`。

15. **Label Smoothing 提升 BLEU 但损害 PPL**：因为它让模型"更不确定"，PPL 惩罚不确定性，BLEU 看是否选对 token `[原论文 §5.4]`。评估指标选错会被误导。

## 11. 开源覆盖缺口（论文/教材未讲透之处）

> 以下是上述四份一手资源**没有讲透或完全没讲**、需要用开源书籍/后续论文补充的点。标注建议补充来源。

### 缺口 1：Pre-LN vs Post-LN 的对比与选择
- **现状**：原论文只给 Post-LN（`LayerNorm(x+Sublayer(x))`）`[原论文 §3.1]`，未讨论 Pre-LN。
- **缺失**：为什么现代大模型改 Pre-LN、Pre-LN 为何训练更稳可省 warmup、DeepNorm 等改进。
- **建议补充**：《Natural Language Processing with Transformers》（Tunstall et al.）或 On Layer Normalization in the Transformer Architecture (Xiong et al., 2020)。

### 缺口 2：现代位置编码全景（ALiBi、NoPE、YaRN、长上下文外推）
- **现状**：原论文给正余弦，RoPE 给旋转，但**长上下文外推（NTK-aware、YaRN）、ALiBi、无位置编码（NoPE）等**都没覆盖。
- **缺失**：长序列外推的实际工程做法。
- **建议补充**：ALiBi (Press et al., 2021)、YaRN (Peng et al., 2023)。

### 缺口 3：KV Cache 与推理优化（Decoder-only 高频考点）
- **现状**：原论文是训练视角，**完全没讲推理时的 KV cache**。
- **缺失**：自回归生成如何复用已算的 K/V 避免重算、PagedAttention、连续批处理（continuous batching）。
- **建议补充**：vLLM (Kwon et al., 2023)、PagedAttention 论文。

### 缺口 4：从零实现一个完整 Transformer 的工程细节
- **现状**：d2l 给了注意力的代码片段 `[d2l 10.3]`，但**完整可运行的 Transformer（含训练循环、beam search）代码**不够系统。
- **缺失**：端到端实现、debug 常见坑。
- **建议补充**：《Hands-On Machine Learning with Scikit-Learn, Keras and TensorFlow》(Géron) 第 16 章，或 The Annotated Transformer (Harvard NLP)。

### 缺口 5：Decoder-only 架构的理论解释与 Scaling Law
- **现状**：原论文是 Encoder-Decoder，**为什么 Decoder-only 主流**论文未论证（本文件 §6.2 是综合业界共识）。
- **缺失**：Scaling Law、涌现能力、Chinchilla 法则。
- **建议补充**：GPT-3 论文、Scaling Laws (Kaplan et al., 2020)、Chinchilla (Hoffmann et al., 2022)。

### 缺口 6：FlashAttention-2/3 与 GPU 编程细节
- **现状**：FlashAttention v1 给了算法和 IO 复杂度 `[Flash §3]`，但 **FlashAttention-2/3 的改进、warp 级并行、Tensor Core 利用**没讲。
- **缺失**：v2/v3 为何更快、CUDA 实现细节。
- **建议补充**：FlashAttention-2 (Dao, 2023)、FlashAttention-3 (Dao, 2024)。

### 缺口 7：Attention 的变体与替代（GQA、MQA、Linear Attention）
- **现状**：原论文是标准 MHA，**Grouped-Query Attention、Multi-Query Attention、Linear Attention** 未涉及。
- **缺失**：大模型推理时如何减少 KV cache（GQA/MQA）。
- **建议补充**：MQA (Shazeer, 2019)、GQA (Ainslie et al., 2023)，LLaMA-2/3 架构解析。

### 缺口 8：为什么 ReLU 后来被 GELU/SwiGLU 取代
- **现状**：原论文 FFN 用 ReLU `[原论文 §3.3]`，但现代 LLM（LLaMA 等）用 SwiGLU。
- **缺失**：GELU/SwiGLU 的动机与效果对比。
- **建议补充**：GELU (Hendrycks & Gimpel, 2016)、SwiGLU (Shazeer, 2020)。

### 缺口 9：注意力可视化与可解释性的深入
- **现状**：原论文附录给了注意力可视化例子（Figure 3-5），但**系统性可解释性方法**未展开。
- **缺失**：attention probing、probing classifier、注意力是否真的"解释"。
- **建议补充**：《Interpretability of Deep Learning》综述，或 Anthropic 的 circuits 工作。

### 缺口 10：BPE/分词对 Transformer 的影响
- **现状**：原论文提到用 BPE（37000 token）/ word-piece（32000）`[原论文 §5.1]`，但**分词如何影响注意力、多语言、中文**未展开。
- **缺失**：SentencePiece、BBPE、分词边界对长程依赖的影响。
- **建议补充**：SentencePiece (Kudo & Richardson, 2018)、严灿平书相关章节。
