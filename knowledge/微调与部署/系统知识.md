## 1. 为什么微调

### 1.1 三种"让模型变好"的路径

- **Prompt 工程**：不改模型权重，靠提示词激发预训练能力。零成本、零延迟，但受限于模型固有能力。
- **RAG（检索增强）**：不改权重，外挂知识库。解决"知识过时/私域知识"，但不改变模型的语气、格式、推理风格。
- **微调（Fine-tuning）**：用梯度更新模型权重，把特定行为/知识"内化"进参数。改变模型本身的能力分布。

**决策原则**：知识更新 → RAG；行为/风格/格式/领域推理能力 → 微调；两者正交、可叠加。

### 1.2 微调能解决什么（来自 LoRA 论文动机）

> "An important paradigm of natural language processing consists of large-scale pre-training on general domain data and adaptation to particular tasks or domains." [来源 1, LoRA 摘要]

微调的本质是把通用基座"适配（adaptation）"到具体下游任务。LoRA 论文指出全参微调的痛点：随着预训练模型变大（GPT-3 175B），"deploying independent instances of fine-tuned models, each with 175B parameters, is prohibitively expensive" [来源 1]。这正是 PEFT 兴起的根本动机。

### 1.3 何时该微调（实战判断）

| 场景 | 建议 |
|------|------|
| 让模型按固定 JSON Schema 输出 | 微调（SFT）小数据即可 |
| 注入私域文档问答 | RAG 优先，必要时微调格式 |
| 改变模型语气/拒绝行为 | SFT + 对齐（DPO/RLHF） |
| 提升数学/代码推理 | 领域 SFT，或 RL（GRPO/RLVR） |
| 仅 1-2 个示例演示 | Prompt 工程 / Few-shot |

## 2. 微调类型：全参 vs PEFT

### 2.1 全参微调（Full Fine-Tuning, FFT）

更新**所有**参数。LoRA 论文定义："During fine-tuning, the model is initialized to pre-trained weights and biases, and all model parameters undergo gradient updates." [来源 1]

- 优点：理论表达能力最强，基线质量上限高。
- 缺点：每个任务都要存一份完整权重副本（与原模型等大）；优化器状态（Adam 的 m、v）再 ×2~3，显存爆炸；多任务部署时存储/切换开销大。

> LoRA 论文量化痛点：全参微调 GPT-3 175B 时，"the new model contains as many parameters as in the original model" [来源 1]，部署多个实例不可行。

### 2.2 PEFT（Parameter-Efficient Fine-Tuning）

PEFT 官方定义："Parameter-Efficient Fine-Tuning (PEFT) methods enable efficient adaptation of large pretrained models to various downstream applications by **only fine-tuning a small number of (extra) model parameters** instead of all the model's parameters. This significantly decreases the computational and storage costs." [来源 6, PEFT README]

主要流派：

| 流派 | 代表方法 | 改什么 | 是否引入推理延迟 |
|------|----------|--------|------------------|
| **Adapter（加层）** | Houlsby Adapter | 在 Transformer block 内插入小 MLP | 是（加深网络） |
| **Soft Prompt / Prefix** | Prefix-Tuning, Prompt-Tuning | 优化输入层激活/prefix | 占用序列长度 |
| **Low-Rank Adaptation** | LoRA, AdaLoRA | 在权重旁加低秩矩阵 | 可合并、零额外延迟 |
| **Bitfit** | Bitfit | 只调 bias | 极小表达力 |
| **IA³** | IA³ | 乘性缩放向量 | 极小 |

> LoRA 论文对前两类方法的批评："existing techniques often introduce inference latency by extending model depth or reduce the model's usable sequence length... Moreover, these methods often fail to match the fine-tuning baselines." [来源 1] —— LoRA 正是为同时解决"延迟"和"质量"而设计。

### 2.3 全参 vs PEFT 显存账本（综合 QLoRA 原文实测）

QLoRA 论文给出了 7B LLaMA 在 FLAN v2 上、batch=1 的逐项显存拆解（极具参考价值）[来源 2]：

| 项 | 显存（7B, batch=1） | 说明 |
|----|---------------------|------|
| LoRA 参数本身 | **26 MB** | 仅占原模型 0.2%，几乎可忽略 |
| LoRA 输入梯度 | 567 MB（gradient checkpointing 后降至 18 MB/seq） | 激活梯度是大头 |
| 4-bit 基座 | 5048 MB | QLoRA 的核心省内存点 |

原文结论："gradient checkpointing is important but also that aggressively reducing the amount of LoRA parameter yields only minor memory benefits." [来源 2] —— 即"激进修剪 LoRA 参数收益有限，关键在于把基座量化到 4-bit"。

由此可推得全参 vs PEFT 的显存模型：

- **全参**：基座（FP16）+ 全部梯度（FP16）+ Adam 优化器状态（m+v，2×FP16）≈ **基座 ×4**。175B 全参 ≈ 175×2×4 = 1400 GB 量级，单机不可行。
- **LoRA（FP16 基座）**：冻结基座（无需梯度/优化器状态）+ 只为 LoRA 参数存梯度/状态 → 显存约为 **基座 + 少量**，但仍要装下整个 FP16 基座（7B≈14GB）。
- **QLoRA（4-bit 基座）**：基座压到 **1/4**（7B≈4GB）+ LoRA 训练开销 → 可在 24/48GB 单卡微调 33B/65B。[来源 2]

### 2.4 PEFT 在多任务部署上的核心价值

LoRA 论文强调的部署模式："A pre-trained model can be shared and used to build many small LoRA modules for different tasks. We can freeze the shared model and efficiently switch tasks by replacing the matrices A and B." [来源 1]

工程含义：生产中可只加载**一份**大基座 + N 个极小的 LoRA 适配器（每个 MB 级），按请求动态切换/路由。vLLM 的 Multi-LoRA serving 正是这一模式的系统化实现（README："Efficient multi-LoRA support for dense and MoE layers"）[来源 7]。

## 3. LoRA 原理与 QLoRA（重点）

### 3.1 LoRA 的核心假设

> "We hypothesize that the change in weights during model adaptation also has a low 'intrinsic rank', leading to our proposed Low-Rank Adaptation (LoRA) approach." [来源 1]

灵感来自 Aghajanyan et al.（2020）：过参数化模型实际"内禀维度（intrinsic dimension）"很低。LoRA 把这一观察从"训练整个模型"推广到"微调时的权重增量 ΔW"。

### 3.2 LoRA 的数学形式（面试必背）

对一个预训练权重矩阵 `W₀ ∈ ℝ^(d×k)`，LoRA 约束其更新用**低秩分解**表示 [来源 1, §4.1]：

```
W₀ + ΔW = W₀ + B A
```

其中：
- `B ∈ ℝ^(d×r)`，`A ∈ ℝ^(r×k)`，秩 `r ≪ min(d, k)`
- 前向：`h = W₀ x + ΔW x = W₀ x + B A x`（W₀ 与 ΔW 共享输入，输出逐元素相加）

**关键细节（全部来自原文）**：
- **初始化**：A 用随机高斯初始化，B 用零初始化，所以训练开始时 `ΔW = BA = 0`，不破坏预训练权重。[来源 1, §4.1]
- **缩放因子**：对 `ΔW x` 再乘 `α/r`，其中 α 是关于 r 的常数。"When optimizing with Adam, tuning α is roughly the same as tuning the learning rate." 原文做法：把 α 固定为第一次尝试的 r 值、不再调，"This scaling helps to reduce the need to retune hyperparameters when we vary r."[来源 1]
- **低秩就够**：原文经验——即使全秩 d 高达 12,288，"a very low rank (i.e., r can be one or two) suffices" for GPT-3 175B。[来源 1]

### 3.3 LoRA 的三大优势（原文）

1. **可共享基座 + 多任务小模块**："A pre-trained model can be shared... We can freeze the shared model and efficiently switch tasks by replacing the matrices A and B." [来源 1]
2. **训练高效、硬件门槛低**："lowers the hardware barrier to entry by up to 3 times when using adaptive optimizers since we do not need to calculate the gradients or maintain the optimizer states for most parameters."[来源 1]
3. **零额外推理延迟（关键）**：部署时可直接算出并存储合并后的 `W = W₀ + BA`，按常规模型推理；切换任务时"recover W₀ by subtracting BA and then adding a different B′A′, a quick operation with very little memory overhead."[来源 1]

> LoRA 摘要的硬指标："LoRA can reduce the number of trainable parameters by 10,000 times and the GPU memory requirement by 3 times" 对比 GPT-3 175B 全参 + Adam。[来源 1]

### 3.4 LoRA 工程用法（PEFT 库）

PEFT README 示例 [来源 6]：

```python
from transformers import AutoModelForCausalLM
from peft import LoraConfig, TaskType, get_peft_model

model = AutoModelForCausalLM.from_pretrained("Qwen/Qwen2.5-3B-Instruct", device_map="cuda")
peft_config = LoraConfig(
    r=16,
    lora_alpha=32,          # 即公式中的 α
    task_type=TaskType.CAUSAL_LM,
    # target_modules=["q_proj", "v_proj", ...]
)
model = get_peft_model(model, peft_config)
model.print_trainable_parameters()
# trainable params: 3,686,400 || all params: 3,089,625,088 || trainable%: 0.1193
```

> 实测：对 `mt0-large` 用 LoRA 只训练 **0.19%** 参数 [来源 6]；对 Qwen2.5-3B，r=16 时 trainable 约 **0.12%**。

加载已训练 LoRA 做推理：

```python
from peft import PeftModel
model = AutoModelForCausalLM.from_pretrained(model_id, device_map="cuda")
model = PeftModel.from_pretrained(model, "qwen2.5-3b-lora")
```

### 3.5 LoRA 超参经验

- **rank r**：常用 4/8/16/32/64。小任务（格式、风格）r=8 即可；领域知识注入可能需 r=32~64。原文经验：很低秩就够。
- **alpha α**：经验法则 `α ≈ 2r`（等效缩放 α/r=2），常见 (r=8,α=16)、(r=16,α=32)。
- **target_modules**：原 LoRA 论文只加在 attention 的 `Wq, Wv`；现代实践常加到全部线性层（`q_proj,k_proj,v_proj,o_proj,gate_proj,up_proj,down_proj`）质量更好（QLoRA 也强调"LoRA on all transformer layers is critical to match 16-bit performance"[来源 2]）。
- **dropout**：LoRA dropout 通常 0.05~0.1。

### 3.6 QLoRA：在 4-bit 量化基座上做 LoRA

QLoRA 摘要："QLoRA... reduces memory usage enough to finetune a 65B parameter model on a single 48GB GPU while preserving full 16-bit finetuning task performance. QLoRA backpropagates gradients through a **frozen, 4-bit quantized pretrained language model** into Low Rank Adapters (LoRA)." [来源 2]

显存账本："QLoRA reduces the average memory requirements of finetuning a 65B parameter model from >780GB of GPU memory to <48GB without degrading the runtime or predictive performance compared to a 16-bit fully finetuned baseline." [来源 2]

#### QLoRA 三大创新

**(a) 4-bit NormalFloat (NF4)** —— 信息论最优的数据类型 [来源 2]：
- 基于分位数量化（quantile quantization），"an information-theoretically optimal data type that ensures each quantization bin has an equal number of values"。
- 关键观察：预训练权重"usually have a zero-centered normal distribution with standard deviation σ"，所以可把所有权重缩放到固定分布，再用标准正态 `N(0,1)` 的 `2^k+1` 个分位构造 k-bit 数据类型。
- 为保证精确表示 0（量化 padding/零值无误差），构造**非对称**类型：负部 `2^(k-1)` 个分位、正部 `2^(k-1)+1` 个，合并去重一个零。结果记为 `NFk`，对零均值正态分布"信息论最优"。

**(b) Double Quantization（双重量化）** [来源 2]：
- 问题：精确 4-bit 量化需要小 blocksize（如 64），而每块的 32-bit 量化常数本身要占 `32/64 = 0.5 bits/parameter` 的额外开销。
- 做法：把第一层量化的 FP32 量化常数 `c₂^FP32` 当作输入，再做一次 8-bit 量化 → 得到 `c₂^FP8` 和第二层常数 `c₁^FP32`。
- 收益："saving an average of about 0.37 bits per parameter (approximately 3 GB for a 65B model)"[来源 2]。

**(c) Paged Optimizers（分页优化器）** [来源 2]：
- 用 NVIDIA unified memory 消除梯度检查点带来的显存尖峰（gradient checkpointing memory spikes），避免 OOM。
- 注意：分页只在处理长序列的小批量时偶发触发，batch=16 时与常规优化器同速。

#### QLoRA 单层前向（原文公式）

```
Y^BF16 = X^BF16 · doubleDequant(c₁^FP32, c₂^k-bit, W^NF4) + X^BF16 · L₁^BF16 · L₂^BF16
```

- 存储数据类型：4-bit NF4；计算数据类型：BF16。
- 反向时：对 4-bit 权重 `∂E/∂W` **不需要**，只需对 LoRA 适配器 `∂E/∂Lᵢ` 求梯度（先反量化到 BF16 再算导数）。[来源 2]

#### QLoRA 关键经验

- "Using LoRA on all transformer layers is critical to match 16-bit performance."[来源 2]
- "data quality is far more important than dataset size"——9k 样本 OASST1 在聊天能力上胜过 450k FLAN v2。[来源 2]
- 最小 Guanaco-7B 部署仅需 5 GB，在 Vicuna 上比 26GB Alpaca 高 20+ 个百分点。[来源 2]

#### LoRA/QLoRA 选型决策

| 维度 | 全参微调 | LoRA | QLoRA |
|------|----------|------|-------|
| 基座精度 | FP16/BF16 | FP16/BF16 | 4-bit NF4 |
| 可训参数 | 100% | ~0.1–1% | ~0.1–1% |
| 显存（7B） | ~28GB+ | ~14GB+ | ~5GB |
| 显存（65B） | 单卡不可行 | 多卡 | 单 48GB 卡可行 [来源 2] |
| 质量上限 | 最高 | 接近全参（覆盖全线性层时） | 接近 16-bit LoRA |
| 推理延迟 | 无额外 | 可合并→无额外 | 基座需反量化（有少量开销） |
| 适合 | 实验室、上限探索 | 生产微调主流 | 单卡、超大模型微调 |

> 经验法则：能上 LoRA 就别全参；显存不够就 QLoRA；要做对齐（DPO/RLHF）通常在 SFT 后的 LoRA/全参上继续训。

#### LoRA 直觉：为什么"低秩"够用

LoRA 原文经验："Using GPT-3 175B as an example, we show that a very low rank (i.e., r can be one or two) suffices even when the full rank (i.e., d) is as high as 12,288." [来源 1]

解读：微调任务的"信息量"远小于预训练——预训练学了整个语言分布，微调只调整到一个窄域。这种"窄调整"在权重空间里对应一个低维子空间，故低秩分解几乎无损。这也解释了为何 r=8 在大多数 SFT 场景已足够，而无需逼近全秩。

## 4. SFT 监督微调

### 4.1 SFT 是什么

SFT = Supervised Fine-Tuning，用 `(指令, 期望输出)` 对，按标准语言建模目标做下一 token 预测的交叉熵损失训练，把基座"教会"按指令格式回答。DPO 论文把 SFT 描述为 RLHF 流水线的第一阶段：[来源 3]

> "RLHF typically begins by fine-tuning a pre-trained LM with supervised learning on high-quality data for the downstream task(s) of interest (dialogue, summarization, etc.), to obtain a model π^SFT." [来源 3, §3]

### 4.2 SFT 数据形态

- **指令数据**：`{"instruction": ..., "input": ..., "output": ...}`（Alpaca 格式）/ 多轮 `messages: [{role, content}]`（ShareGPT/OpenAI 格式）。
- **损失掩码（loss masking）**：只在"assistant 输出"token 上算 loss，prompt/user 部分掩掉。这是 SFT 与普通 LM 预训练的关键差异。

### 4.3 SFT 训练要点

- 通常用 LoRA/QLoRA 而非全参，2.5k~10k 高质量样本即可显著改变行为。
- 学习率：LoRA 1e-4~3e-4；全参 1e-5~2e-5。epoch 通常 1~3，过多会过拟合 + 灾难性遗忘。
- 重要陷阱：**灾难性遗忘（catastrophic forgetting）**——纯领域数据微调会让通用能力退化。缓解：混合通用数据、控制 epoch、用 LoRA（冻结基座天然缓解）。

### 4.4 SFT → 对齐的标准链路

基座模型 → SFT（学会格式/指令）→ 偏好对齐（RLHF/DPO，学会"哪个回答更好"）。DPO 论文的三阶段正是 `SFT → Reward Modelling → RL` [来源 3, §3]。

## 5. 对齐：RLHF / DPO / GRPO（重点）

### 5.1 RLHF（PPO）三阶段（来自 DPO 论文综述）

DPO 论文系统描述了标准 RLHF [来源 3, §3]：

1. **SFT**：得到 `π^SFT`。
2. **奖励建模（Reward Modelling）**：
   - 用 `π^SFT` 对 prompt x 采样一对答案 `(y₁,y₂)`，人工标注偏好 `y_w ≻ y_l | x`。
   - 假设存在潜在奖励 `r*(y,x)`，用 **Bradley-Terry (BT) 模型**建模偏好分布：
     `p*(y₁≻y₂|x) = σ(r*(x,y₁) − r*(x,y₂))`（σ 是 logistic 函数）。
   - 训练奖励模型 `r_φ(x,y)`，用负对数似然（二分类）。
3. **RL 优化**：在奖励上加 KL 约束，防止策略漂离参考模型：

   ```
   max_{πθ} E[r_φ(x,y)] − β·D_KL(πθ ‖ π_ref)
   ```

   其中 β 控制偏离基线 `π_ref`（=π^SFT）的程度。由于离散生成不可微，标准做法把奖励改写为 `r(x,y) = r_φ(x,y) − β(log πθ(y|x) − log π_ref(y|x))`，用 **PPO** 最大化。[来源 3]

### 5.2 RLHF/PPO 的痛点

DPO 摘要直接点名："RLHF is a complex and often unstable procedure, first fitting a reward model... and then fine-tuning the large unsupervised LM using reinforcement learning." [来源 3]

具体难点：
- 需要 4 个模型同时在显存里：actor（πθ）、reference（π_ref）、reward、critic/value（PPO 估值用）。显存极大。
- 训练中要"从 LM 采样"（on-policy rollouts），慢。
- PPO 超参敏感，奖励 hacking、模式坍塌等不稳定。

### 5.3 DPO：用分类损失直接优化偏好（重点）

#### 核心思想

DPO 摘要："we introduce a new parameterization of the reward model in RLHF that enables extraction of the corresponding optimal policy **in closed form**, allowing us to solve the standard RLHF problem with only a simple classification loss." [来源 3]

> 一句话：**"Your Language Model is Secretly a Reward Model"**——语言模型本身就能表示隐式奖励，无需单独训练奖励模型、无需 RL。

#### 推导（原文 §4）

1. KL 约束奖励最大化目标的最优解有闭式形式 [来源 3]：
   `π*(y|x) = (1/Z(x)) · π_ref(y|x) · exp(r(x,y)/β)`，其中 `Z(x)` 是配分函数。
2. 反解出用策略表示奖励（reparameterization）：
   `r(x,y) = β · log(π*(y|x)/π_ref(y|x)) + β·log Z(x)`
3. 代入 Bradley-Terry 偏好模型，**配分函数 Z(x) 相消**，得到只用策略表示的偏好概率。
4. 对参数化策略 `πθ` 写最大似然，得 **DPO 损失**：

   ```
   L_DPO(πθ; π_ref) = − E_{(x,y_w,y_l)~D} [ log σ( β·log(πθ(y_w|x)/π_ref(y_w|x)) − β·log(πθ(y_l|x)/π_ref(y_l|x)) ) ]
   ```

   其中定义**隐式奖励** `r̂_θ(x,y) = β·log(πθ(y|x)/π_ref(y|x))`。[来源 3]

#### 梯度的直觉（原文）

> "the gradient of L_DPO increases the likelihood of the preferred completions y_w and decreases the likelihood of dispreferred completions y_l. Importantly, the examples are weighed by **how much higher the implicit reward model r̂_θ rates the dispreferred completions**, scaled by β, i.e., how incorrectly the implicit reward model orders the completions, accounting for the strength of the KL constraint." [来源 3]

> 警告（原文实测）：去掉这个动态重要性权重的"naïve version"会让模型退化（degenerate）[来源 3, Appendix Table 3]。这是 DPO 与简单"拉大好答案/压低坏答案 logprob"的关键差别。

#### DPO 流程（原文）

1. 对每个 prompt x 采样一对 `(y₁,y₂) ∼ π_ref`，人工标注偏好，构造离线数据集 `D = {(xⁱ, y_wⁱ, y_lⁱ)}`。
2. 优化 `πθ` 最小化 `L_DPO`，给定 `π_ref` 与 β。
3. 实践：`π_ref = π^SFT`（若已有 SFT 模型）；若没有，先用偏好回答做 MLE 得到 `π_ref`。[来源 3]

#### DPO vs PPO 对比

| 维度 | PPO-RLHF | DPO |
|------|----------|-----|
| 需要奖励模型？ | 是 | 否（隐式） |
| 需要 RL/采样？ | 是 | 否 |
| 损失 | RL 目标 | 二元交叉熵（分类） |
| 显存模型数 | 4（actor/ref/reward/critic） | 2（πθ/π_ref，可共享） |
| 稳定性 | 复杂、敏感 | 稳定 |
| 质量原文结论 | — | "exceeds PPO-based RLHF in ability to control sentiment... matches or improves response quality in summarization and single-turn dialogue while being substantially simpler to implement and train" [来源 3] |

#### DPO 关键超参 β

- β 小 → 允许策略偏离参考模型更多，更像 RLHF 强约束弱（学习更激进，风险：退化）。
- β 大 → 强 KL 约束，贴近参考模型，变化小。
- 经验 β ∈ [0.1, 0.5]，常用 0.1。

### 5.4 GRPO：去掉 value 网络的 PPO 变体（DeepSeek）

> 经 WebSearch 在 arXiv:2402.03300 (DeepSeekMath) 验证 [来源 11]。

GRPO（Group Relative Policy Optimization）核心：
- 是 PPO 的变体，**消除 value/critic 网络**（PPO 里最大的显存与训练负担之一）。
- **组相对优势**：对每个 prompt 采样一组 G 个输出，用组内奖励的均值 μ 与标准差 σ 归一化得到优势 `Aᵢ = (rᵢ − μ)/σ`，替代 PPO 的 `A = R − V(s)`（V 是学出来的价值函数）。
- **规则奖励**：奖励常来自**确定性规则**（如数学答案正确性检查），而非学习的奖励模型，降低 reward hacking 风险。
- **KL 正则**：在当前策略与参考策略间加 KL 惩罚，保持稳定。
- 应用：DeepSeek-R1-Zero / DeepSeek-R1 的推理能力核心训练算法。

RLHF/DPO/GRPO 选型：通用对齐/易实现 → DPO；数学/代码等可验证奖励 → GRPO（RLVR）；研究/最强上限 → PPO-RLHF（最重）。

### 5.5 对齐方法族谱与选型决策（综合）

```
            人类/AI 偏好数据
                  │
        ┌─────────┼──────────┐
        ▼         ▼          ▼
     RLHF/PPO    DPO        KTO/IPO/SimPO
   (显式奖励+RL) (隐式奖励,  (DPO 变体,
                 分类损失)    改损失/去 ref)
        │
        ▼  规则可验证奖励
      GRPO (去 value 网络, 组相对优势)
        │
        ▼
     RLVR / DeepSeek-R1
```

- **DPO**：偏好对 `(y_w, y_l)`，二元交叉熵，需参考模型 π_ref。[来源 3]
- **DPO 衍生**：
  - **IPO**（Identity Preference Optimization）：修正 DPO 在数据充分时的过拟合。
  - **KTO**（Kahneman-Tversky Optimization）：只需"好/坏"二元标签（不需成对），用前景理论的价值函数。
  - **SimPO**：去掉参考模型 π_ref，用归一化奖励 + 目标 margin，更省显存。
- **GRPO/RLVR**：当奖励可被规则验证（数学答案、代码单测）时，跳过偏好标注，直接用规则奖励 + GRPO。[来源 11]

### 5.6 DPO 实现要点（基于原文 + 工程实践）

1. **数据**：每条样本 `(prompt, chosen, rejected)`。chosen/rejected 通常是同一 prompt 下两个回答。
2. **参考模型**：用 SFT 模型冻结作 π_ref，前向时同时跑 πθ 与 π_ref 算 logprob。
3. **损失**：对 chosen 和 rejected 分别算 `log(πθ/π_ref)`，套 β 做 DPO 损失。
4. **显存优化**：π_ref 可用 LoRA 的 base 权重共享、或用 8-bit 量化加载；πθ 用 LoRA 训练。常见做法是"一份 SFT 基座 + LoRA 当 πθ，基座冻结当 π_ref"。
5. **超参**：β=0.1、lr=5e-7~5e-6（比 SFT 小一个量级）、epoch 1~3。
6. **陷阱**：原文警告 naïve 损失（去掉重要性权重）会退化 [来源 3]——务必用完整 sigmoid 形式，别只做"抬 chosen 压 rejected"。

## 6. 量化：INT8/INT4、AWQ、GPTQ、GGUF（重点）

### 6.1 量化的基本概念（来自 QLoRA 论文）

> "Quantization is the process of discretizing an input from a representation that holds more information to a representation with less information. It often means taking a data type with more bits and converting it to fewer bits, for example from 32-bit floats to 8-bit Integers." [来源 2]

为利用低比特类型的全范围，通常用输入元素的绝对最大值做归一化再映射。例如 FP32→INT8 `[-127,127]` [来源 2]。

### 6.2 量化分类

- **按时机**：PTQ（Post-Training Quantization，训练后一次性量化，无需重训）vs QAT（Quantization-Aware Training，训练中模拟量化）。
- **按对象**：weight-only（只量化权重，激活仍 FP16，最常用）、weight+activation（两端都量化，更省但更难）。
- **按比特**：INT8、INT4、INT3，乃至 sub-4bit。

### 6.3 INT8 / bitsandbytes

- 最常见入门量化。LLM.int8()（Dettmers 2022）发现：少数"离群值（outlier）"特征列幅度极大，混合精度——离群列保持 FP16，其余 INT8，避免精度崩塌。
- bitsandbytes 库支持，HF transformers 一行 `load_in_8bit=True`。
- 显存约为 FP16 的一半，质量损失通常可忽略。

### 6.4 GPTQ（一次性、二阶信息/Hessian）

> 经 WebSearch 在 arXiv:2210.17323 验证 [来源 9]。

- **原理**：基于 OBQ（Optimal Brain Quantization），逐列量化权重，量化一个权重后，用**近似二阶信息（Hessian 矩阵）**补偿剩余 FP 权重，最小化量化引入的输出误差。
- 用 Hessian 衡量每个权重方向的重要性，把误差集中到不重要的方向。
- **一次性后训练（one-shot）**：只需少量校准数据（~128 条），即可把几百亿参数模型压到 4-bit，质量损失很小。
- **代表实现**：AutoGPTQ。常见 `gptq-4bit` 模型。
- 注意：GPTQ 是**weight-only** 的逐列后训练量化，依赖校准集。

### 6.5 AWQ（Activation-aware Weight Quantization）

> 经 WebSearch 在 arXiv:2306.00978（MLSys 2024）验证 [来源 8]。

- **核心观察**："not all weights in an LLM are equally important. **Protecting only 1% salient weights** can greatly reduce quantization error." —— 保护约 1% 的"显著权重"就能大幅降低量化误差。
- **显著权重的判据**：不看权重本身幅度，而看**对应激活的幅度**（activation magnitude）——激活大的通道对应的权重更重要。
- **做法**：per-channel scaling，给显著通道的权重更大保护（保持 FP16 或更高精度），其余 INT3/INT4。**不依赖反向传播**，是硬件友好的 weight-only 量化。
- **优势**：比 GPTQ 更快（无需逐列二阶补偿）、对齐硬件更好，INT4 下几乎无损。
- **代表实现**：mit-han-lab/llm-awq；vLLM 原生支持 AWQ。

### 6.6 GGUF / llama.cpp（端侧量化）

- **GGUF**（GPT-Generated Unified Format，前 GGML）是 llama.cpp 生态的二进制格式，把权重 + 元数据打包，支持 **k-quants** 命名体系：`Q2_K, Q3_K_M, Q4_K_M, Q5_K_M, Q6_K, Q8_0` 等。
- 特点：K-quant 是**分块混合精度**（attention 等关键层用更高比特、其余更低），在 CPU/Apple Silicon 上推理友好。
- 端侧/本地部署主流格式，配合 Ollama、llama.cpp 使用。

### 6.7 量化方法选型速查

| 方法 | 时机 | 数据 | 适合场景 | 实现生态 |
|------|------|------|----------|----------|
| bitsandbytes INT8/4 | PTQ / 训练 | 校准 | 训练（QLoRA）、快速加载 | HF transformers |
| GPTQ-4bit | PTQ 一次性 | ~128 校准 | 服务部署、显存极省 | AutoGPTQ, vLLM |
| AWQ-4bit | PTQ 无反传 | 少量校准 | 服务部署、速度优先 | llm-awq, vLLM |
| GGUF (K-quants) | PTQ | - | CPU/Mac 端侧 | llama.cpp, Ollama |
| FP8 | PTQ / 训练 | - | 新 GPU（H100/Ada） | vLLM, TransformerEngine |

> vLLM README 列出的原生量化支持 [来源 7]："Quantization: FP8, MXFP8/MXFP4, NVFP4, INT8, INT4, GPTQ/AWQ, GGUF, compressed-tensors, ..." —— 即一个推理引擎统一承接几乎所有主流量化格式。

### 6.8 量化深入：AWQ vs GPTQ vs NF4 对比（综合来源 2/8/9）

| 维度 | NF4 (QLoRA) | GPTQ | AWQ |
|------|-------------|------|-----|
| 用途 | **训练**（QLoRA 基座） | **推理**部署 | **推理**部署 |
| 数据类型构造 | 标准正态分位（信息论最优）[来源 2] | 朴素 INT4 + 二阶补偿 | 朴素 INT4 + 显著通道保护 |
| 依赖反向传播？ | 是（用于训练） | 否（一次性 PTQ） | 否（一次性 PTQ） |
| 校准集 | 不需要（固定数据类型） | 需要（~128 条，算 Hessian） | 需要（算激活幅度） |
| 保护对象 | 整体（用最优分位） | 逐列误差补偿 | 1% 显著权重（按激活） |
| 推理速度 | 慢（需反量化训练） | 快 | 最快（硬件友好） |
| 典型精度损失 | 几乎无（训练恢复 16bit）[来源 2] | 很小 | 很小 |
| 适用 | 微调 | 通用部署 | 通用部署、对速度敏感 |

关键区分：**NF4 是为"训练时存储/计算分离"设计**（存 4-bit、算 BF16），而 **GPTQ/AWQ 是为"推理时一次性压缩"设计**。QLoRA 训完部署时通常也会再转成 GPTQ/AWQ/GGUF 做推理。

### 6.9 量化实战：从 FP16 模型到 INT4 部署的典型路径

1. **训练阶段**：拿 FP16/BF16 基座 → QLoRA 微调（NF4 存储 + LoRA 训练）[来源 2]。
2. **合并**：把训好的 LoRA 合并回基座 `W = W₀ + BA`（可先反量化基座到 FP16 再合并）。
3. **推理量化**：对合并后的 FP16 模型用 GPTQ 或 AWQ 做一次性 INT4 量化（用校准集）。
4. **部署**：用量化后的 `.gptq-4bit` / `.awq` 权重 + vLLM/Ollama 服务。或转 GGUF 走 llama.cpp 端侧。

> 注意：直接对 4-bit 基座合并 LoRA 再部署也可以（vLLM 支持），但精度与速度权衡需实测。

## 7. 推理优化：KV Cache、Flash Attention、PagedAttention、vLLM、Speculative Decoding（重点）

### 7.1 自回归生成的瓶颈（来自 vLLM/PagedAttention 论文）

> "This sequential generation process makes the workload **memory-bound**, underutilizing the computation power of GPUs and limiting the serving throughput." [来源 5]

LLM 逐 token 生成，是**访存受限（memory-bound）**的——算力富余、显存读写是瓶颈。优化方向几乎都围绕"减少访存、提高显存利用率"。

### 7.2 KV Cache

- 生成第 t 个 token 时，前面 t−1 个 token 的 K、V 不变，缓存复用避免重算 → 把每步 O(t) 复杂度降到 O(1)（相对前缀）。
- **代价**：KV cache 占显存巨大且动态增长。PagedAttention 论文实测：13B 模型在 A100 上，KV cache 占了**近 30%** 的显存 [来源 5]。
- **问题**：传统框架要求张量连续存放，预分配最大长度 → 严重碎片化。下文 PagedAttention 即解决此问题。

### 7.3 Flash Attention（重点，来自原文）

#### 要解决的问题

> "We argue that a missing principle is making attention algorithms **IO-aware** — accounting for reads and writes between levels of GPU memory." [来源 4, 摘要]

现代 GPU 算力增长快于显存带宽，Transformer 大多数算子**被访存（HBM accesses）瓶颈化**。标准 attention 要把 N×N 的注意力矩阵完整写/读 HBM，代价巨大。

#### 两大技术

1. **Tiling（分块）**：把 Q/K/V 切成块，在 GPU 片上 SRAM 内增量式做 softmax 归约，**不把 N×N 注意力矩阵物化到 HBM**。"computing the softmax reduction without access to the whole input... splitting the input into blocks and making several passes over input blocks, thus incrementally performing the softmax reduction (also known as tiling)." [来源 4]
   - 外层循环遍历 K/V 块载入 SRAM，内层循环遍历 Q 块，把 attention 输出写回 HBM。
2. **Recomputation（反向重算）**：反向时**不存**中间注意力矩阵，而是用前向保存的 softmax 归一化因子在片上重算 attention。"which is faster than the standard approach of reading the intermediate attention matrix from HBM." [来源 4]

#### IO 复杂度（原文证明）

- FlashAttention HBM 访问：`O(N²d²M⁻¹)`（M = SRAM 大小，d = 头维度）
- 标准 attention HBM 访问：`Ω(Nd + N²)`
- 在典型 d、M 下，FlashAttention 比标准 attention **最多少 ~9× 的 HBM 访问**，并证明"no exact attention algorithm can asymptotically improve on the number of HBM accesses over all SRAM sizes"（下界）。[来源 4]

#### 关键性质

- **精确（exact）**：结果与标准 attention 数学等价，不是近似。
- **更快更省显存**：GPT-2 上 attention 计算快 **7.6×**，显存**随序列长度线性**（而非 N²）。[来源 4]
- 即便重算多了 FLOPs，因大幅减少 HBM 访问，整体仍更快。
- 可扩展为 **block-sparse FlashAttention**（近似），比任何已知近似 attention 都快。

### 7.4 PagedAttention / vLLM（重点，来自原文）

#### 要解决的问题

PagedAttention 论文直指 KV cache 管理的低效 [来源 5, §1]：

- "existing systems... store the KV cache of a request in **contiguous memory space**."
- KV cache 特性：动态增长收缩、生命周期与长度事先未知 → 传统连续分配导致**内部碎片 + 外部碎片**。
- 实测："only **20.4% - 38.2%** of the KV cache memory is used to store the actual token states in the existing systems."（即 60%+ 被浪费）[来源 5]
- 此外，并行采样/beam search 等多输出场景，多个序列**无法共享** KV cache。

#### PagedAttention 思想：OS 虚拟内存 + 分页

> "we propose PagedAttention, an attention algorithm inspired by the operating system's (OS) solution to memory fragmentation and sharing: **virtual memory with paging**." [来源 5]

- "PagedAttention divides the request's KV cache into **blocks**, each of which can contain the attention keys and values of a **fixed number of tokens**." [来源 5]
- "the blocks for the KV cache are **not necessarily stored in contiguous space**."
- 类比：**blocks ↔ pages，tokens ↔ bytes，requests ↔ processes**。
- 用小的、按需分配的块 → 把内部碎片降到"最后一个块"内；块表（block table）做逻辑→物理映射。
- 块可被多序列共享（copy-on-write）→ 并行采样/beam search 的前缀共享零拷贝。

#### vLLM 整体收益（原文）

> "vLLM improves the throughput of popular LLMs by **2-4×** with the same level of latency compared to the state-of-the-art systems, such as FasterTransformer and Orca." [来源 5, 摘要]

vLLM 提供：(1) KV cache 近零浪费；(2) 灵活的块内/跨请求 KV 共享。

#### Continuous Batching（连续批处理）

- 传统 static batching：一个 batch 里所有序列要等最长的生成完才能释放 → 短序列占着资源空等。
- **Continuous batching（也叫 iteration-level / dynamic batching）**：每一步（iteration）都重新组 batch——刚结束的请求移出、新请求插入，长尾请求不再拖累整体。
- vLLM 配合 PagedAttention 实现：因 KV cache 按 block 管理而非连续，请求进出无需搬运内存，开销极小。
- vLLM README 进一步列出配套优化 [来源 7]：chunked prefill、prefix caching、piecewise/full CUDA graphs。

### 7.5 Speculative Decoding（投机解码）

> 经 WebSearch 在 arXiv:2211.17192（Leviathan, Kalman, Matias）验证 [来源 10]。

**动机**：自回归逐 token 慢。能否"猜"多个 token 再批量验证？

**机制**：
1. **Draft model（小、快）**：先投机地生成一段候选 token。
2. **Target model（大、慢）**：用一个前向并行验证这批候选 token。
3. **Accept/Reject（接受/拒绝）**：
   - 若候选 token 与 target 分布一致 → 接受；
   - 不一致 → 拒绝，并从**残差分布（residual distribution）**采样一个修正 token。
4. **关键保证**：输出分布与"只用 target model 采样"**完全相同（mathematically identical）**——无质量损失。
5. **加速**：典型 2~3×，前提是 draft 比 target 快得多（1~10×），且两者词表/分布相近。

实现：vLLM 支持 n-gram、suffix、**EAGLE**、DFlash 等投机解码策略 [来源 7]。

### 7.7 FlashAttention 细节：为什么精确又省（深入原文）

精确性的来源——**online softmax（在线 softmax）**：

标准 softmax `softmax(x)_i = exp(x_i − m) / Σ exp(x_j − m)`（减最大值 m 保数值稳定）。问题：分母需要对全行求和，必须看到整个 N 维。

FlashAttention 的做法：把 Q/K/V 分块，**逐块累加** softmax 的 running max `m` 和 running sum `l`。每处理一个新 K/V 块，更新全局 m、l 并重缩放已累加的输出。这样无需物化整个 N×N 矩阵，却得到与标准 attention **逐位相同**的结果。这就是原文说的 "incrementally performing the softmax reduction (also known as tiling)"[来源 4]。

反向重算的取舍：标准实现要存 N×N 的 attention 概率矩阵供反向用，显存 O(N²)。FlashAttention 反向时只用前向存的 m、l 在 SRAM 重算 attention，显存 O(N)。代价是反向多了 FLOPs，但因 GPU 是访存受限、这些重算都在片上极快，净收益为正——"which is faster than the standard approach of reading the intermediate attention matrix from HBM"[来源 4]。

硬件背景（原文，A100 为例）[来源 4]：
- HBM：40–80 GB，带宽 1.5–2.0 TB/s
- 片上 SRAM：每 SM 192 KB，比 HBM 快一个量级以上
- "compute speed has out-paced memory speed"——所以减少 HBM 访问是提速关键

### 7.8 PagedAttention 细节：碎片化数据与共享（深入原文）

PagedAttention 论文给出碎片化的硬数据 [来源 5]：现有系统（FasterTransformer、Orca）中"only 20.4% - 38.2% of the KV cache memory is used to store the actual token states"——即 60%+ 显存被内部/外部碎片浪费。

碎片来源（原文分析）：
1. **内部碎片**：为请求预分配最大长度（如 2048 tokens）的连续块，但实际生成可能只用了几百 token，剩下全空占。
2. **外部碎片**：不同请求预分配大小不一，块与块之间产生无法利用的缝隙。
3. **无法共享**：并行采样（一次请求出多个序列）、beam search 中，多个序列有大量公共前缀，但传统连续存储无法共享 KV，重复存储。

PagedAttention 的解法（OS 类比）[来源 5]：
- KV cache 按 **block（页）** 分配，每块存固定数量 token 的 K/V。
- 块**物理上非连续**，用 **block table（页表）** 做逻辑块号→物理块号映射。
- 按需分配：生成新 token 才申请新块，结束即回收 → 内部碎片只剩"最后一个未填满块"。
- **共享**：多个序列前缀相同时，block table 指向同一组物理块；写分歧时用 **copy-on-write**（COW）复制块。→ 并行采样/beam search 显存大幅下降。

### 7.9 Continuous Batching vs Static Batching（图解）

```
Static Batching（传统）：
请求A: [prefill][gen gen gen gen DONE ──────────────── 等 B]
请求B: [prefill][gen gen gen gen gen gen gen gen gen DONE]
→ A 早完成却要陪 B 跑完整个 batch，GPU 空转。

Continuous Batching（vLLM）：
step1: A B C      step2: A B C      step3: A B D      step4: B D E ...
→ 每步重组：A 完成即移出，新请求 D/E 插入，GPU 持续满载。
配合 PagedAttention，请求进出无需搬运 KV 显存（块化管理）。
```

为何只有 PagedAttention 能高效做 continuous batching：传统连续 KV 下，移出/插入请求需要搬运大块显存，开销巨大；分页后只需改 block table，O(1) 元数据操作。[来源 5]

## 10. 面试考点（带答案）

**Q1：LoRA 为什么能减少可训练参数？写出公式。**
A：基于"权重增量 ΔW 内禀低秩"假设，把全秩更新约束为低秩分解 `W = W₀ + BA`，其中 `B∈ℝ^(d×r)`、`A∈ℝ^(r×k)`，`r ≪ min(d,k)`，冻结 W₀ 只训 A、B。前向 `h = W₀x + BAx`。可训练参数从 d·k 降到 r·(d+k)。原文 GPT-3 175B 上"减少 10,000× 可训练参数、3× GPU 显存"。[来源 1]

**Q2：LoRA 的 A 和 B 怎么初始化？为什么？**
A：A 随机高斯、B 初始化为零 → 训练开始 `BA=0`，不破坏预训练权重，相当于恒等启动。前向再加缩放 `α/r`。[来源 1]

**Q3：LoRA 为什么"无额外推理延迟"？**
A：部署时可直接合并 `W = W₀ + BA` 存为单一矩阵，按常规模型推理，没有额外分支。这与 adapter（加深网络）和 prefix-tuning（占序列长度）形成对比。[来源 1]

**Q4：QLoRA 的三大创新是什么？**
A：(1) **4-bit NormalFloat (NF4)**——对零均值正态分布信息论最优的 4-bit 数据类型；(2) **Double Quantization**——对量化常数再做 8-bit 量化，省 ~0.37 bits/参数；(3) **Paged Optimizers**——用 NVIDIA 统一内存消除梯度检查点的显存尖峰。[来源 2]

**Q5：QLoRA 如何在 4-bit 上反传？**
A：存储用 NF4、计算用 BF16。用到一个权重张量时反量化到 BF16 做矩阵乘；反传只需对 LoRA 适配器 `∂E/∂Lᵢ` 求梯度（通过反量化后的导数），4-bit 权重的梯度不更新。[来源 2]

**Q6：RLHF 三阶段？为什么不稳定？**
A：SFT → 奖励建模（BT 模型）→ RL（PPO，带 KL 约束）。不稳定来自：需 4 个模型同显存（actor/ref/reward/critic）、要 on-policy 采样、PPO 超参敏感、奖励 hacking。[来源 3]

**Q7：DPO 的损失长什么样？为什么不需要奖励模型？**
A：`L_DPO = −E[log σ( β log(πθ(y_w|x)/π_ref(y_w|x)) − β log(πθ(y_l|x)/π_ref(y_l|x)) )]`。由 KL 约束奖励最大化最优解的闭式形式反解出"奖励 = β·log(πθ/π_ref)"，把 BT 偏好模型改写为仅含策略的形式，配分函数相消，于是直接用二元交叉熵拟合偏好——策略本身即隐式奖励模型。[来源 3]

**Q8：DPO 里 β 的作用？**
A：β 控制 KL 约束强度（偏离参考模型 π_ref 的程度）。隐式奖励 `r̂ = β·log(πθ/π_ref)`，β 小→允许更大偏离、学得更激进；β 大→强约束、变化小。常用 0.1。[来源 3]

**Q9：FlashAttention 为什么快？是近似吗？**
A：精确（非近似）。通过 tiling 在 SRAM 内增量做 softmax，避免 N×N 矩阵读写 HBM；反向用归一化因子重算而非读中间矩阵。HBM 访问从 `Ω(Nd+N²)` 降到 `O(N²d²/M)`，最多少 9×。GPT-2 上快 7.6×，显存随序列线性。[来源 4]

**Q10：PagedAttention 解决什么？原理？**
A：解决 KV cache 在连续内存下的内部/外部碎片（实测仅 20.4%–38.2% 有效利用）。借鉴 OS 虚拟内存分页：KV 分成固定 token 数的 block，非连续存储，blocks↔pages、tokens↔bytes、requests↔processes；按需分配 + 块表映射 + 块共享（COW）。vLLM 因此吞吐提升 2–4×。[来源 5]

**Q11：Continuous Batching 是什么？**
A：每次 iteration 重新组 batch——结束的请求移出、新请求插入，长短请求不互相阻塞。配合 PagedAttention 的非连续 KV，请求进出无需搬内存。[来源 5]/[来源 7]

**Q12：AWQ 和 GPTQ 的区别？**
A：GPTQ 基于 OBQ，逐列量化后用 Hessian 二阶信息补偿剩余权重，需校准集、一次性。AWQ 发现"1% 显著权重"主导误差，**按激活幅度**（而非权重幅度）识别显著通道，用 per-channel scaling 保护它们，无需二阶补偿、更硬件友好。两者都是 PTQ weight-only INT4。[来源 8]/[来源 9]

**Q13：Speculative Decoding 改变输出分布吗？**
A：不变。draft model 投机生成候选，target model 并行验证，接受一致的、从残差分布修正不一致的，数学上保证与仅用 target 采样分布完全相同，典型 2–3× 加速。[来源 10]

**Q14：GRPO 相比 PPO 省了什么？**
A：去掉了 value/critic 网络。用组内采样 G 个输出的奖励均值/标准差做组相对优势，替代 PPO 的 `A=R−V(s)`；奖励常用规则（如答案正确性）而非学习模型，加 KL 正则。[来源 11]

## 11. 易错点与陷阱

1. **LoRA 默认 target_modules**：早期只挂 q/v proj；现在要质量好应覆盖全部线性层（QLoRA 实测："LoRA on all transformer layers is critical to match 16-bit performance"）。[来源 2]
2. **α/r 缩放别忘**：LoRA 前向是 `(α/r)·BAx`，不是 `BAx`。HF PEFT 里 `lora_alpha` 即 α。
3. **LoRA 合并后不可动态切多任务**：若把 BA 吸收进 W 消除延迟，则同一 batch 不同任务用不同 LoRA 就难批处理（需不合并、动态选 LoRA 模块）。[来源 1]
4. **DPO 不是"简单拉大好/压坏 logprob"**：去掉动态重要性权重的 naïve 版会让模型退化。损失里隐式奖励差值外还套 sigmoid，权重随"坏答案被隐式奖励高估的程度"动态调整。[来源 3]
5. **DPO 的 π_ref 来源**：理想 `π_ref = π^SFT`；若直接用偏好数据集但没 SFT 模型，应先用偏好回答做 MLE 初始化 π_ref，缓解分布偏移。[来源 3]
6. **FlashAttention ≠ 近似注意力**：它是精确的，常被误当成 Linformer/线性注意力那类近似。block-sparse 版才是近似。[来源 4]
7. **量化不是无损**：INT4 在某些任务（数学、长尾知识）有可见退化；离群值大的模型需 AWQ/GPTQ 这种保护显著权重的方法，而非朴素 round-to-nearest。[来源 8]/[来源 9]
8. **GPTQ 需要校准集**：纯 weight-only 但量化顺序依赖激活统计，没有校准数据质量会掉。
9. **KV cache 是显存大头**：13B 模型近 30% 显存给 KV；长上下文下 KV 增长惊人，PagedAttention/量化 KV（如 FP8 KV）是关键。[来源 5]
10. **Continuous Batching ≠ Dynamic Batching**：前者是 token/iteration 级（每步重组），后者通常是请求级（攒一批再发）。前者吞吐更高。
11. **QLoRA 显存省在基座，不在梯度**：原文实测 7B 上 LoRA 参数仅 26MB，但输入梯度 567MB（gradient checkpointing 后 18MB），4-bit 基座 5048MB——即"激进修剪 LoRA 参数收益有限，关键是 4-bit 基座"。[来源 2]
12. **Speculative Decoding 依赖 draft/target 分布相近**：若两者差异大，接受率低，反而因多跑 draft 而变慢。
13. **灾难性遗忘**：纯窄域 SFT 会丢通用能力；混合通用数据、控 epoch、用 LoRA 缓解。
14. **DPO 的偏好数据要来自 π_ref（最好是 π^SFT）分布**：用更强模型生成的偏好对训更弱 SFT 模型会有分布不匹配问题。[来源 3]
15. **GGUF 的 Q4_K_M 不是均匀 4-bit**：是分块混合精度，attention 等关键层更高比特，别按"平均 4bit"线性估算质量。
