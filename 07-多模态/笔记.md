## 2. 核心知识串联

### 2.1 什么是多模态（Multimodal）

- **定义**：让模型同时理解和处理多种模态的信息（文本、图像、音频、视频等）。
- **为什么重要**：现实世界是多模态的，纯文本 LLM 无法理解图片、无法听语音。
- **核心挑战**：不同模态的表示空间不同，需要**对齐**（alignment）到统一空间。

### 2.2 ViT（Vision Transformer）— 视觉编码器基础

- **核心思想**：把图像切成固定大小的 patch（如 16×16），每个 patch 展平后当作一个 "token"，送入标准 Transformer Encoder。
- **流程**：
  1. 图像 → 切成 N 个 patch（如 224×224 图像切成 14×14 = 196 个 patch）
  2. 每个 patch 线性投影为向量（类似 word embedding）
  3. 加上位置编码 + [CLS] token
  4. 送入 Transformer Encoder → 输出 [CLS] 向量作为图像表示
- **关键点**：ViT 证明了 Transformer 不仅能处理文本，也能处理图像，且在大数据集上效果超越 CNN。
- **面试高频**：ViT 和 CNN 的区别？ViT 的 patch 大小对性能的影响？

### 2.3 CLIP（Contrastive Language-Image Pre-training）

- **核心思想**：用**对比学习**把图像和文本映射到同一个向量空间。
- **架构**：
  - 图像编码器（ViT 或 ResNet）→ 图像向量
  - 文本编码器（Transformer）→ 文本向量
  - 训练目标：匹配的图文对相似度高，不匹配的相似度低
- **训练数据**：4 亿图文对（从互联网收集）
- **Zero-shot 能力**：不需要微调，直接用文本描述做分类（如 "a photo of a cat"）
- **应用场景**：图文检索、零样本分类、图像生成引导（Stable Diffusion 用 CLIP 做文本编码）
- **面试高频**：CLIP 的对比学习 loss 是什么？CLIP 为什么能做 zero-shot？

### 2.4 多模态大模型（LLaVA / GPT-4V / Gemini）

- **核心架构**：视觉编码器 + 投影层 + LLM
  ```
  图像 → ViT → 视觉 token → 投影层(线性/MLP) → 与文本 token 拼接 → LLM → 回答
  ```
- **LLaVA 训练两阶段**：
  1. **预训练**：冻结 ViT 和 LLM，只训投影层（学习视觉-语言对齐）
  2. **指令微调**：解冻 LLM，用视觉问答数据微调（学习遵循指令）
- **GPT-4V / Gemini**：闭源，架构未公开，但能力更强（多图理解、OCR、空间推理）
- **面试高频**：多模态大模型怎么把图像信息传给 LLM？投影层的作用？

### 2.5 多模态应用场景

| 场景 | 说明 | 技术要点 |
|---|---|---|
| 图文检索 | 用文本搜图片，或用图片搜文本 | CLIP 嵌入 + 向量检索 |
| 视觉问答（VQA） | 看图回答问题 | LLaVA / GPT-4V |
| 文档理解 | 理解 PDF、表格、图表 | OCR + 多模态 LLM |
| 图像生成 | 文本描述生成图像 | Stable Diffusion（CLIP 文本编码 + U-Net + VAE） |
| 视频理解 | 理解视频内容、生成摘要 | 视频帧采样 + 多模态 LLM |

## 3. 面试速查清单

1. **ViT**：把图像切 patch 当 token，用 Transformer Encoder 处理；大数据集上超越 CNN。
2. **CLIP**：对比学习对齐图文空间；4 亿图文对训练；支持 zero-shot 分类。
3. **对比学习 loss**：InfoNCE loss，batch 内正样本相似度最大化，负样本最小化。
4. **多模态大模型架构**：ViT + 投影层 + LLM；关键是投影层把视觉 token 映射到 LLM 的输入空间。
5. **LLaVA 两阶段**：先训投影层（对齐），再微调 LLM（指令跟随）。
6. **CLIP vs 多模态 LLM**：CLIP 只做对齐/匹配，不能生成文本；多模态 LLM 能理解图像并生成回答。
7. **图像生成**：Stable Diffusion = CLIP 文本编码 + U-Net 去噪 + VAE 解码。

## 4. 易错/陷阱

- ViT 不是"用 CNN 提特征再给 Transformer"，而是直接把 patch 当 token（纯 Transformer）。
- CLIP 的 zero-shot 不是"不需要训练"，而是"不需要在目标任务上微调"（预训练用了 4 亿数据）。
- 多模态大模型的投影层很关键，不是简单拼接图像和文本就行。
- LLaVA 第一阶段冻结 ViT 和 LLM，只训投影层——参数量很小但效果显著。
- GPT-4V 能力强但闭源，面试中要能讲清开源方案（LLaVA）的原理。
